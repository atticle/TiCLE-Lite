# 손 동작으로 제어하기

## 학습 목표

- MediaPipe Hands로 손 랜드마크를 검출하고 좌표를 추출할 수 있다.
- 손가락 굽힘 여부로 제스처(gesture)를 판별하는 규칙을 설계할 수 있다.
- 제스처 인식 결과를 MQTT로 전송해 TiCLE Lite를 제어할 수 있다.

---

## 이 장의 도전 문제

> 손을 펼치면 LED가 켜지고, 주먹을 쥐면 꺼지며, 손가락을 하나씩 펴는 것으로 밝기가 달라지도록 만들어 보자.

이 장의 핵심은 MediaPipe 라이브러리 사용 방법이 아니다. **"손의 기하학적 정보를 제스처 판별 규칙으로 변환하는 설계"** 가 핵심이다.

## 사전 지식. MediaPipe 손 랜드마크

MediaPipe Hands는 손에서 21개의 랜드마크(landmark) 좌표를 추출한다. 각 랜드마크는 손 위의 특정 위치이다.

```
손가락 관절 번호 (각 손가락):

  엄지: 1-2-3-4
  검지: 5-6-7-8
  중지: 9-10-11-12
  약지: 13-14-15-16
  소지: 17-18-19-20
  손바닥: 0
```

손가락이 **펴졌는지** 판별하는 방법은 간단하다. 손가락 끝(tip)의 y좌표가 두 번째 관절(pip)의 y좌표보다 **위에** 있으면 펴진 것이다. (화면 좌표계에서 y가 작을수록 위쪽)

```python
# 검지 펴짐 여부: 끝(8)이 두번째관절(6)보다 위
is_index_up = lm[8].y < lm[6].y
```

## 미션 1. 손 랜드마크를 화면에 표시하라

### 문제

PC 카메라에서 손을 인식해 21개의 랜드마크 점을 화면에 그려라.

### 코드

파일 이름: `m1_hand_landmarks.py` (PC에서 실행)

```python
# m1_hand_landmarks.py  — PC에서 실행
import cv2
import mediapipe as mp

mp_hands = mp.solutions.hands
mp_draw  = mp.solutions.drawing_utils

hands = mp_hands.Hands(max_num_hands=1, min_detection_confidence=0.7)
cap   = cv2.VideoCapture(0)

while True:
    ret, frame = cap.read()
    if not ret:
        break

    frame_rgb = cv2.cvtColor(frame, cv2.COLOR_BGR2RGB)
    result    = hands.process(frame_rgb)

    if result.multi_hand_landmarks:
        for hand_lm in result.multi_hand_landmarks:
            # 랜드마크 연결선 그리기
            mp_draw.draw_landmarks(frame, hand_lm, mp_hands.HAND_CONNECTIONS)

            # 각 랜드마크에 번호 표시
            h, w, _ = frame.shape
            for idx, lm in enumerate(hand_lm.landmark):
                cx, cy = int(lm.x * w), int(lm.y * h)
                cv2.putText(frame, str(idx), (cx, cy),
                            cv2.FONT_HERSHEY_PLAIN, 0.8, (0,255,255), 1)

    cv2.imshow("손 랜드마크", frame)
    if cv2.waitKey(1) & 0xFF == ord('q'):
        break

cap.release()
cv2.destroyAllWindows()
```


### 관찰하고 실험해 보자

- 손을 펼치면 21개 점이 모두 나타나는가?
- 손을 움직이면 랜드마크가 따라오는가?
- 검지(8번)의 좌표는 주먹을 쥘 때와 펼 때 y값이 어떻게 달라지는가?

### 알게 된 것

각 랜드마크의 좌표는 0.0~1.0 사이의 정규화된 값이다. 실제 픽셀 좌표로 변환하려면 프레임의 너비(`w`)와 높이(`h`)를 곱해야 한다.

## 미션 2. 손가락이 펴졌는지 판별하라

### 문제

각 손가락이 펴졌는지(1) 접혔는지(0)를 5자리 이진수로 출력하라.

예: 검지+중지만 펴면 `[0, 1, 1, 0, 0]` 출력

### 생각해 보기

각 손가락의 끝(tip)과 두 번째 관절(pip) 번호:

| 손가락 | tip | pip |
|--------|-----|-----|
| 엄지 | 4 | 2 |
| 검지 | 8 | 6 |
| 중지 | 12 | 10 |
| 약지 | 16 | 14 |
| 소지 | 20 | 18 |

엄지는 x축으로 판별한다(왼손/오른손에 따라 다름). 나머지 4개 손가락은 y축으로 판별한다.

### 코드

파일 이름: `m2_finger_count.py` (PC에서 실행)

```python
# m2_finger_count.py  — PC에서 실행
import cv2
import mediapipe as mp

mp_hands = mp.solutions.hands
hands    = mp_hands.Hands(max_num_hands=1, min_detection_confidence=0.7)
cap      = cv2.VideoCapture(0)

# (tip_idx, pip_idx) 쌍 (엄지 제외)
FINGER_PAIRS = [(8,6), (12,10), (16,14), (20,18)]

def get_finger_states(lm_list):
    """[엄지, 검지, 중지, 약지, 소지] 펴짐 여부를 반환한다."""
    lm  = lm_list
    states = []

    # 엄지: x축으로 판별 (오른손 기준)
    thumb_up = lm[4].x < lm[3].x
    states.append(1 if thumb_up else 0)

    # 나머지 4손가락
    for tip, pip in FINGER_PAIRS:
        states.append(1 if lm[tip].y < lm[pip].y else 0)

    return states

while True:
    ret, frame = cap.read()
    if not ret:
        break

    rgb    = cv2.cvtColor(frame, cv2.COLOR_BGR2RGB)
    result = hands.process(rgb)

    if result.multi_hand_landmarks:
        lm     = result.multi_hand_landmarks[0].landmark
        states = get_finger_states(lm)
        count  = sum(states)
        print(f"손가락: {states}  펴진 수: {count}")

    cv2.imshow("손가락 판별", frame)
    if cv2.waitKey(1) & 0xFF == ord('q'):
        break

cap.release()
cv2.destroyAllWindows()
```


## 미션 3. 제스처를 정의하고 MQTT로 발행하라

### 문제

주먹(`[0,0,0,0,0]`), 펼침(`[1,1,1,1,1]`), 검지만(`[0,1,0,0,0]`)을 감지해 각각 `"off"`, `"on"`, `"point"` 문자열을 MQTT로 발행하라.

### 코드

파일 이름: `m3_gesture_mqtt.py` (PC에서 실행)

```python
# m3_gesture_mqtt.py  — PC에서 실행
import cv2
import mediapipe as mp
import paho.mqtt.client as mqtt

BROKER = "test.mosquitto.org"
TOPIC  = "ticle/hand/gesture"

GESTURE_MAP = {
    (0,0,0,0,0): "off",
    (1,1,1,1,1): "on",
    (0,1,0,0,0): "point",
    (0,1,1,0,0): "peace",
    (0,0,0,0,1): "pinky",
}

mp_hands     = mp.solutions.hands
FINGER_PAIRS = [(8,6),(12,10),(16,14),(20,18)]

def get_states(lm):
    s = [1 if lm[4].x < lm[3].x else 0]
    for tip, pip in FINGER_PAIRS:
        s.append(1 if lm[tip].y < lm[pip].y else 0)
    return tuple(s)

client = mqtt.Client()
client.connect(BROKER, 1883)

hands     = mp_hands.Hands(max_num_hands=1, min_detection_confidence=0.7)
cap       = cv2.VideoCapture(0)
SKIP      = 5
count     = 0
last_sent = ""

while True:
    ret, frame = cap.read()
    if not ret:
        break

    count += 1
    if count % SKIP == 0:
        rgb    = cv2.cvtColor(frame, cv2.COLOR_BGR2RGB)
        result = hands.process(rgb)

        if result.multi_hand_landmarks:
            lm      = result.multi_hand_landmarks[0].landmark
            states  = get_states(lm)
            gesture = GESTURE_MAP.get(states, "unknown")

            if gesture != "unknown" and gesture != last_sent:
                client.publish(TOPIC, gesture)
                print(f"발행: {states} → {gesture}")
                last_sent = gesture

    cv2.imshow("제스처 MQTT", frame)
    if cv2.waitKey(1) & 0xFF == ord('q'):
        break

cap.release()
cv2.destroyAllWindows()
```

```sh
python m3_gesture_mqtt.py
```

## 미션 4. TiCLE Lite가 제스처로 반응하게 하라

### 문제

MQTT로 수신한 제스처 문자열에 따라 LED를 제어하라.

| 수신 제스처 | LED 동작 |
|-----------|---------|
| `"on"` | 전체 초록색 |
| `"off"` | 전체 끄기 |
| `"point"` | 지정 패턴 표시 |
| `"peace"` | 무지개 패턴 |

### 코드

파일 이름: `m4_gesture_led.py` (TiCLE Lite에서 실행)

```python
# m4_gesture_led.py
import network, utime, math
from ws2812 import WS2812Matrix
from umqtt.simple2 import MQTTClient

SSID   = "여기에 Wi-Fi 이름"
PW     = "여기에 Wi-Fi 비밀번호"
BROKER = "test.mosquitto.org"
TOPIC  = b"ticle/hand/gesture"

sta = network.WLAN(network.STA_IF)
sta.active(True)
sta.connect(SSID, PW)
while not sta.isconnected():
    utime.sleep_ms(200)

m = WS2812Matrix([(0, 1)])

def fill(r, g, b):
    for x in range(16):
        for y in range(16):
            m[x, y].value = (r, g, b)
    m.update()

def hsv_to_rgb(h):
    """h: 0~360 → (r,g,b)"""
    h = h % 360
    s, v = 1.0, 0.4
    c = v * s
    x = c * (1 - abs((h/60) % 2 - 1))
    r1,g1,b1 = [(c,x,0),(x,c,0),(0,c,x),(0,x,c),(x,0,c),(c,0,x)][int(h/60)]
    m_= v - c
    return int((r1+m_)*255), int((g1+m_)*255), int((b1+m_)*255)

def pattern_point():
    m.clear()
    for i in range(16):
        m[8, i].value = (0, 0, 200)
        m[i, 8].value = (0, 0, 200)
    m.update()

def pattern_peace():
    for step in range(20):
        for x in range(16):
            for y in range(16):
                r, g, b = hsv_to_rgb((x*22 + step*18) % 360)
                m[x, y].value = (r, g, b)
        m.update()
        utime.sleep_ms(60)

cmd_queue = []

def on_message(topic, msg, retain, dup):
    gesture = msg.decode().strip()
    cmd_queue.append(gesture)
    print("제스처:", gesture)

client = MQTTClient(b"ticle-hand", BROKER)
client.set_callback(on_message)
client.connect()
client.subscribe(TOPIC)
print("제스처 대기 중")

while True:
    client.check_msg()

    if cmd_queue:
        g = cmd_queue.pop(0)
        if g == "on":
            fill(0, 150, 0)
        elif g == "off":
            m.clear(); m.update()
        elif g == "point":
            pattern_point()
        elif g == "peace":
            pattern_peace()

    utime.sleep_ms(50)
```

## 최종 프로젝트. 손가락 수로 밝기를 제어하라

PC에서 `m3_gesture_mqtt.py`를 수정해 손가락 수(0~5)를 `"bright:N"` 형태로 발행하도록 바꾼다.

TiCLE Lite에서는 `bright:N`을 수신해 밝기를 0~150 사이의 5단계로 조절한다.

**PC 코드 수정 부분** (`m3_gesture_mqtt.py`에서):

```python
# 손가락 수를 밝기 단계로 발행
finger_count = sum(states)
bright_msg   = f"bright:{finger_count}"
if bright_msg != last_sent:
    client.publish(TOPIC, bright_msg)
    last_sent = bright_msg
```

**TiCLE Lite 수신 처리 추가** (`m4_gesture_led.py`에서):

```python
elif g.startswith("bright:"):
    n   = int(g.split(":")[1])
    val = n * 30   # 0~5 → 0~150
    fill(val, val, val)
```
