## 1. Qt와 PySide6

### Qt란 무엇인가

Qt(큐트)는 노르웨이 기업 Trolltech에서 1991년에 시작된 크로스 플랫폼 GUI 프레임워크이다. 현재는 **The Qt Company**가 관리한다. Windows, macOS, Linux에서 동일한 코드가 동작하는 것이 가장 큰 장점이다.

Qt는 본래 C++로 작성됐지만 Python 바인딩을 통해 Python에서도 쓸 수 있다. Python 바인딩에는 두 가지가 있다.

| 라이브러리 | 관리 주체 | 라이선스 | 특징 |
|-----------|----------|---------|------|
| **PyQt6** | Riverbank Computing | GPL / 상용 | 역사 길고 자료 많음 |
| **PySide6** | The Qt Company | LGPL | 공식 바인딩, Qt6 기반 |

이 교재에서는 **PySide6**를 사용한다. The Qt Company가 직접 관리하는 **공식 Python 바인딩**이고, LGPL 라이선스이므로 학교 교육 및 상업적 프로젝트 모두에서 무료로 사용할 수 있다.

### PyQt6와 PySide6의 차이

API 구조는 거의 동일하지만 세부 문법에서 차이가 있다.

| 항목 | PyQt6 | PySide6 |
|------|-------|---------|
| 시그널 클래스 | `pyqtSignal` | `Signal` |
| 슬롯 장식자 | `pyqtSlot` | `Slot` |
| 앱 실행 | `app.exec_()` | `app.exec()` |
| 임포트 패키지 | `PyQt6.QtWidgets` | `PySide6.QtWidgets` |

이후 나오는 모든 코드는 PySide6 문법을 기준으로 한다.

## 3. 첫 번째 창을 만들어 보자

### 가장 단순한 창

파일 이름: `hello_window.py`

```python
# hello_window.py
import sys
from PySide6.QtWidgets import QApplication, QWidget, QLabel, QVBoxLayout

app    = QApplication(sys.argv)   # 앱 객체는 반드시 먼저 만든다

window = QWidget()
window.setWindowTitle("첫 번째 창")
window.setFixedSize(300, 150)

label  = QLabel("안녕하세요, PySide6!")
layout = QVBoxLayout()
layout.addWidget(label)
window.setLayout(layout)

window.show()
sys.exit(app.exec())   # 이벤트 루프 시작
```

```sh
python hello_window.py
```

이 코드에서 알아야 할 것:

1. `QApplication(sys.argv)` — 앱의 심장이다. 모든 창을 관리한다. 반드시 가장 먼저 만들어야 한다.
2. `QWidget` — **위젯(Widget)** 이란 화면에 표시되는 모든 UI 구성 요소를 통틀어 부르는 말이다. 버튼, 레이블, 슬라이더, 입력창, 체크박스가 모두 위젯이다. `QWidget`은 그 모든 위젯의 공통 조상(기반 클래스)으로, 그 자체만으로는 아무 내용도 없는 빈 창이 된다.
3. `QVBoxLayout` — **레이아웃(Layout)** 이란 위젯들을 창 안에 어디에, 어떻게 배치할지를 결정하는 규칙이다. `QVBoxLayout`(Vertical Box Layout)은 위젯을 위에서 아래로 세로로 쌓는다.
4. `app.exec()` — 이벤트 루프를 시작한다. 사용자가 창을 닫을 때까지 여기서 기다린다.

### 클래스로 구조화하기

코드가 길어지면 클래스 방식으로 구성하는 것이 훨씬 관리하기 쉽다. 앞으로는 모두 이 방식을 사용한다.

```python
# hello_class.py
import sys
from PySide6.QtWidgets import QApplication, QWidget, QLabel, QVBoxLayout

class MyWindow(QWidget):
    def __init__(self):
        super().__init__()
        self.setWindowTitle("클래스 방식")
        self.setFixedSize(300, 150)

        label  = QLabel("클래스로 만든 창")
        layout = QVBoxLayout(self)
        layout.addWidget(label)

if __name__ == "__main__":
    app = QApplication(sys.argv)
    win = MyWindow()
    win.show()
    sys.exit(app.exec())
```

`if __name__ == "__main__"` 블록은 이 파일을 직접 실행할 때만 코드가 동작하도록 한다. 다른 파일에서 `import`할 때는 창이 열리지 않는다.

## 4. 레이아웃 시스템

화면에 위젯 하나를 놓는 것은 간단하다. 그런데 버튼 3개, 슬라이더 2개, 레이블 5개를 보기 좋게 배치하려면 어떻게 해야 할까? 픽셀 좌표(x, y)로 직접 지정할 수도 있지만, 창 크기가 조금만 바뀌어도 배치가 어긋나고 코드 수정이 매우 번거로워진다.

**레이아웃(Layout)** 이 이 문제를 해결한다. 레이아웃은 "이 위젯들을 세로로 쌓아라", "저 위젯들은 가로로 나열하라"처럼 배치 규칙을 정의한다. 위젯을 레이아웃에 추가하기만 하면, 창 크기가 바뀌어도 위젯이 규칙에 따라 자동으로 재배치된다.

### QVBoxLayout — 세로 정렬

`addWidget()`으로 위젯을 추가한 순서대로 위에서 아래로 쌓인다. 가장 기본적인 레이아웃으로, 설정 화면처럼 항목이 위아래로 나열되는 구조에 사용한다.

```python
layout = QVBoxLayout(self)
layout.addWidget(QLabel("위"))
layout.addWidget(QLabel("아래"))
```

### QHBoxLayout — 가로 정렬

`addWidget()` 순서대로 왼쪽에서 오른쪽으로 나란히 배치된다. 버튼 여러 개를 한 줄에 늘어놓거나, 레이블과 입력창을 같은 줄에 배치할 때 자주 사용한다.

```python
layout = QHBoxLayout(self)
layout.addWidget(QLabel("왼쪽"))
layout.addWidget(QLabel("오른쪽"))
```

### QGridLayout — 격자 정렬

행(row)과 열(column) 번호를 지정해 원하는 칸에 위젯을 배치한다. 계산기처럼 격자 형태 UI를 만들 때 적합하다. `addWidget(위젯, 행, 열)` 형태로 사용한다.

```python
from PySide6.QtWidgets import QGridLayout, QPushButton

layout = QGridLayout(self)
layout.addWidget(QPushButton("(0,0)"), 0, 0)
layout.addWidget(QPushButton("(0,1)"), 0, 1)
layout.addWidget(QPushButton("(1,0)"), 1, 0)
layout.addWidget(QPushButton("(1,1)"), 1, 1)
```

### 레이아웃 중첩

VBox 안에 HBox를, HBox 안에 VBox를 넣는 것처럼 레이아웃을 겹쳐 쓸 수 있다. 실제 앱의 대부분은 이 중첩 방식으로 복잡한 UI를 구성한다. `addLayout()`으로 레이아웃 안에 다른 레이아웃을 추가한다.

```python
# 상단: 레이블 / 하단: 버튼 두 개를 가로로
from PySide6.QtWidgets import (QWidget, QLabel, QPushButton,
                                QVBoxLayout, QHBoxLayout, QApplication)
import sys

class NestedLayout(QWidget):
    def __init__(self):
        super().__init__()
        self.setWindowTitle("레이아웃 중첩")
        self.resize(300, 120)

        main_layout = QVBoxLayout(self)
        main_layout.addWidget(QLabel("상단 레이블"))

        btn_layout = QHBoxLayout()
        btn_layout.addWidget(QPushButton("왼쪽 버튼"))
        btn_layout.addWidget(QPushButton("오른쪽 버튼"))
        main_layout.addLayout(btn_layout)   # 레이아웃 안에 레이아웃

if __name__ == "__main__":
    app = QApplication(sys.argv)
    win = NestedLayout()
    win.show()
    sys.exit(app.exec())
```

> **참고: Qt Designer — 마우스로 UI를 설계하는 도구**
>
> PySide6에는 코드를 직접 쓰지 않고 마우스로 위젯을 끌어다 놓아 UI를 설계하는 **Qt Designer** 도구가 포함되어 있다. 터미널에서 `pyside6-designer`를 실행하면 시각적 편집기가 열린다. 완성한 UI는 `.ui` 파일(XML 형식)로 저장되며, Python 코드에서 아래와 같이 불러올 수 있다.
>
> ```python
> from PySide6.QtUiTools import QUiLoader
> from PySide6.QtCore import QFile
>
> loader = QUiLoader()
> file   = QFile("my_ui.ui")
> file.open(QFile.OpenModeFlag.ReadOnly)
> widget = loader.load(file)
> widget.show()
> ```
>
> 위젯 종류와 배치가 복잡해질수록 Qt Designer가 훨씬 빠르고 직관적이다. 이 교재에서는 코드의 구조와 원리를 손으로 익히는 것이 목표이므로 코드 방식을 사용하지만, 본격적인 앱을 만들 때는 Qt Designer 활용을 적극 권장한다.

## 5. 주요 위젯과 시그널/슬롯

### 시그널과 슬롯이란 무엇인가

GUI 프로그래밍의 핵심은 "사용자가 무언가를 했을 때 특정 함수가 실행되는" 구조이다. Qt에서는 이것을 **시그널(Signal)** 과 **슬롯(Slot)** 이라는 개념으로 구현한다.

```
[위젯] ──시그널──→ 연결(connect) ──→ [함수(슬롯)]
버튼 클릭이 발생         ↕           실행되고 싶은 함수
```

- **시그널**: 사건이 발생했다는 신호 (클릭, 값 변경, 텍스트 입력 등)
- **슬롯**: 시그널을 받으면 실행되는 함수
- **connect()**: 시그널과 슬롯을 연결하는 메서드

```python
button.clicked.connect(my_function)   # 버튼이 클릭되면 my_function 실행
```

### QPushButton (버튼)

가장 기본적인 입력 위젯이다. 클릭하면 `clicked` 시그널이 발생한다. 아래 예제는 버튼을 누를 때마다 클릭 횟수를 화면에 표시한다.

```python
# button_demo.py
import sys
from PySide6.QtWidgets import QApplication, QWidget, QLabel, QPushButton, QVBoxLayout

class ButtonDemo(QWidget):
    def __init__(self):
        super().__init__()
        self.setWindowTitle("버튼 예제")
        self.count = 0

        self.label  = QLabel("아직 안 눌렸음")
        button      = QPushButton("클릭!")
        button.clicked.connect(self.on_click)   # 시그널 연결

        layout = QVBoxLayout(self)
        layout.addWidget(self.label)
        layout.addWidget(button)

    def on_click(self):
        self.count += 1
        self.label.setText(f"{self.count}번 클릭됨")

if __name__ == "__main__":
    app = QApplication(sys.argv)
    win = ButtonDemo()
    win.show()
    sys.exit(app.exec())
```

### QSlider (슬라이더)

손잡이를 드래그해 연속적인 값을 조절하는 위젯이다. 밝기, 볼륨, 모터 속도처럼 범위가 있는 값을 입력받을 때 유용하다. 손잡이를 움직이는 즉시 `valueChanged` 시그널이 발생하므로 실시간 제어에 적합하다.

```python
from PySide6.QtWidgets import QSlider
from PySide6.QtCore import Qt

slider = QSlider(Qt.Orientation.Horizontal)   # 가로 슬라이더
slider.setRange(0, 255)                        # 최솟값, 최댓값
slider.setValue(128)                           # 초기값

slider.valueChanged.connect(lambda v: print("값:", v))
```

### QLineEdit (텍스트 입력)

한 줄 텍스트를 입력받는 위젯이다. `setPlaceholderText()`로 입력 전에 흐리게 표시할 안내 문구를 설정할 수 있다. Enter 키를 누르면 `returnPressed` 시그널이 발생하며, `text()`로 현재 입력된 문자열을 읽는다.

```python
from PySide6.QtWidgets import QLineEdit

edit = QLineEdit()
edit.setPlaceholderText("여기에 입력하세요")
edit.returnPressed.connect(lambda: print("입력:", edit.text()))
```

### QLCDNumber (숫자 표시)

전광판(LCD)처럼 생긴 숫자 표시 위젯이다. `setDigitCount()`로 자릿수를 지정하고 `display()`로 값을 갱신한다. 시리얼이나 MQTT로 받은 센서값을 직관적으로 보여줄 때 사용한다.

```python
from PySide6.QtWidgets import QLCDNumber

lcd = QLCDNumber()
lcd.setDigitCount(5)
lcd.display(12345)
```

### QProgressBar (진행 막대)

숫자 값을 채워진 막대 그래프로 나타내는 위젯이다. `setRange()`로 최솟값·최댓값을 자유롭게 설정할 수 있다. ADC 센서값(0~65535)처럼 넓은 범위의 값을 한눈에 시각화할 때 유용하다.

```python
from PySide6.QtWidgets import QProgressBar

bar = QProgressBar()
bar.setRange(0, 65535)   # ADC 16비트 범위
bar.setValue(32000)
```

### 위젯 정리표

| 위젯 클래스 | 주요 시그널 | 역할 |
|-------------|------------|------|
| `QLabel` | (없음) | 읽기 전용 텍스트 / 이미지 표시 |
| `QPushButton` | `clicked` | 클릭 버튼 |
| `QLineEdit` | `textChanged`, `returnPressed` | 한 줄 텍스트 입력 |
| `QSlider` | `valueChanged` | 범위 값 조절 |
| `QComboBox` | `currentIndexChanged` | 드롭다운 선택 |
| `QLCDNumber` | (없음) | 숫자 표시판 |
| `QProgressBar` | (없음) | 진행/크기 막대 |
| `QCheckBox` | `stateChanged` | 체크박스 |

## 6. 커스텀 시그널 만들기

위젯이 제공하는 시그널 외에 직접 시그널을 정의할 수도 있다. 클래스 변수로 선언하고 `emit()`으로 발송한다.

```python
from PySide6.QtCore import QObject, Signal

class Sensor(QObject):
    value_ready = Signal(int)    # int를 전달하는 시그널 정의

    def read(self):
        value = 12345            # 센서 읽기 (예시)
        self.value_ready.emit(value)   # 시그널 발송

sensor = Sensor()
sensor.value_ready.connect(lcd.display)   # 시그널 → LCD 업데이트
sensor.read()
```

시그널에 여러 타입을 전달하거나 문자열을 전달할 수도 있다.

```python
data_received = Signal(str)       # 문자열 전달
error_occurred = Signal(str, int) # 문자열과 정수 동시 전달
```

## 7. QTimer — 주기적으로 실행하기

### time.sleep()을 절대 쓰지 말아야 하는 이유

```python
# 잘못된 코드 - GUI가 완전히 멈춘다!
while True:
    val = read_sensor()
    label.setText(str(val))
    time.sleep(0.5)   # ← 이 동안 클릭, 창 이동 등 모든 것이 멈춤
```

GUI는 **이벤트 루프**(`app.exec()`)가 계속 돌아야 사용자 입력에 반응한다. `time.sleep()`으로 블록하면 이벤트 루프가 잠겨 창이 응답하지 않는다.

해결책이 `QTimer`이다. "N 밀리초마다 이 함수를 실행해 달라"고 이벤트 루프에 부탁하는 방식이다. `sleep()`과 달리 이벤트 루프는 계속 돌아간다.

### QTimer 사용법

```python
# timer_demo.py
import sys
from PySide6.QtWidgets import QApplication, QWidget, QLabel, QVBoxLayout
from PySide6.QtCore import QTimer
import datetime

class ClockWindow(QWidget):
    def __init__(self):
        super().__init__()
        self.setWindowTitle("디지털 시계")
        self.setFixedSize(250, 80)

        self.time_label = QLabel("00:00:00")
        self.time_label.setStyleSheet("font-size: 36px; font-weight: bold;")

        layout = QVBoxLayout(self)
        layout.addWidget(self.time_label)

        # 1000ms(1초)마다 update_time 호출
        timer = QTimer(self)
        timer.timeout.connect(self.update_time)
        timer.start(1000)
        self.update_time()   # 처음 한 번 즉시 실행

    def update_time(self):
        now = datetime.datetime.now().strftime("%H:%M:%S")
        self.time_label.setText(now)

if __name__ == "__main__":
    app = QApplication(sys.argv)
    win = ClockWindow()
    win.show()
    sys.exit(app.exec())
```

```sh
python timer_demo.py
```

`QTimer`의 동작 방식:

```
이벤트 루프
    ↓
    타이머 만료(1000ms)
    ↓
    update_time() 호출
    ↓
    label 업데이트
    ↓
    다시 이벤트 루프로 복귀
```

### 한 번만 실행하기

```

## 8. QThread — 백그라운드 작업

### 왜 QThread가 필요한가

QTimer는 "짧고 빠른 작업"에 적합하다. 시리얼 포트 열기, 파일 다운로드, MQTT 수신 대기처럼 **시간이 걸리거나 블로킹되는 작업**은 QTimer로 처리할 수 없다. 이런 작업을 별도 스레드에서 실행하는 것이 `QThread`이다.

```
메인 스레드                     워커 스레드
─────────────────              ──────────────────
이벤트 루프 (GUI 반응)          시리얼/MQTT 수신 대기
  ↕                             ↕
시그널/슬롯으로 통신 ←──────── 데이터 도착 시 Signal.emit()
```

### 핵심 규칙

> **다른 스레드에서 GUI 위젯을 직접 수정하면 안 된다.**
> Signal을 통해 메인 스레드에 데이터를 전달하고, 메인 스레드가 GUI를 업데이트해야 한다.

### Worker 패턴

가장 권장되는 방식은 **QThread를 상속**하는 것이다.

```python
from PySide6.QtCore import QThread, Signal

class SerialWorker(QThread):
    data_received = Signal(str)   # 데이터가 도착하면 이 시그널로 전달
    error_occurred = Signal(str)

    def __init__(self, port, baudrate):
        super().__init__()
        self.port     = port
        self.baudrate = baudrate
        self._running = True

    def run(self):
        """별도 스레드에서 실행된다. 이 안에서 GUI 수정 금지."""
        import serial
        try:
            ser = serial.Serial(self.port, self.baudrate, timeout=1)
            while self._running:
                line = ser.readline().decode("utf-8", errors="ignore").strip()
                if line:
                    self.data_received.emit(line)   # 메인 스레드로 전달
            ser.close()
        except Exception as e:
            self.error_occurred.emit(str(e))

    def stop(self):
        self._running = False
        self.wait()   # 스레드가 완전히 종료될 때까지 기다린다
```

## 실습 1. 시리얼 통신으로 TiCLE Lite를 제어하라

### 구조

TiCLE Lite를 PC에 USB 케이블로 연결하면 시리얼 포트(COM 포트)로 보인다. REPL 대신 커스텀 프로토콜을 사용하면 PC GUI 프로그램이 보드와 직접 데이터를 주고받을 수 있다.

```
PC (PySide6 GUI)  ←──시리얼(USB)──→  TiCLE Lite (MicroPython)
슬라이더 → "bright:80\n"  →  LED 밝기 설정
쿼리 → "cds?\n"         →  "cds:48320\n"  → LCD 숫자 표시
```

### TiCLE Lite 쪽 코드

먼저 TiCLE Lite에 다음 코드를 올린다. USB 시리얼로 명령을 받아 처리한다.

파일 이름: `serial_server.py` (TiCLE Lite에서 실행)

```python
# serial_server.py  — TiCLE Lite에서 실행
import sys
from machine import Pin, ADC
from ws2812 import WS2812Matrix

m   = WS2812Matrix([(0, 1)])
cds = ADC(Pin(28))
vr  = ADC(Pin(27))

def fill(r, g, b):
    for x in range(16):
        for y in range(16):
            m[x, y].value = (r, g, b)
    m.update()

# stdin/stdout을 시리얼 통신으로 활용
while True:
    try:
        line = sys.stdin.readline().strip()
        if not line:
            continue

        if line.startswith("bright:"):
            val = min(int(line.split(":")[1]), 150)
            fill(val, val, val)
            sys.stdout.write("ok\n")

        elif line.startswith("color:"):
            parts = line.split(":")[1].split(",")
            r, g, b = [min(int(p), 150) for p in parts]
            fill(r, g, b)
            sys.stdout.write("ok\n")

        elif line == "clear":
            m.clear()
            m.update()
            sys.stdout.write("ok\n")

        elif line == "cds?":
            sys.stdout.write(f"cds:{cds.read_u16()}\n")

        elif line == "vr?":
            sys.stdout.write(f"vr:{vr.read_u16()}\n")

        else:
            sys.stdout.write(f"err:unknown:{line}\n")

    except Exception as e:
        sys.stdout.write(f"err:{e}\n")
```

### PC GUI 코드 (미션 2): 센서값 수신 + 밝기 제어를 동시에

센서 데이터를 읽으려면 **주기적으로 쿼리를 보내고 응답을 기다려야** 한다. 이것은 블로킹 동작이므로 `QThread`를 사용한다.

파일 이름: `step2_dashboard.py` (PC에서 실행)

```python
# step2_dashboard.py  — PC에서 실행
import sys
import serial
from PySide6.QtWidgets import (QApplication, QWidget, QLabel, QSlider,
                                QProgressBar, QVBoxLayout, QHBoxLayout,
                                QGroupBox)
from PySide6.QtCore import Qt, QThread, Signal, QTimer

PORT     = "COM3"      # 자신의 포트 번호로 바꾼다
BAUDRATE = 115200

# ── 워커 스레드 ──────────────────────────────────────
class SensorWorker(QThread):
    cds_updated = Signal(int)
    vr_updated  = Signal(int)
    error_msg   = Signal(str)

    def __init__(self, port, baudrate):
        super().__init__()
        self.port      = port
        self.baudrate  = baudrate
        self._running  = True
        self._ser      = None

    def run(self):
        try:
            self._ser = serial.Serial(self.port, self.baudrate, timeout=1)
            while self._running:
                # CdS 쿼리
                self._ser.write(b"cds?\n")
                resp = self._ser.readline().decode("utf-8", errors="ignore").strip()
                if resp.startswith("cds:"):
                    try:
                        self.cds_updated.emit(int(resp.split(":")[1]))
                    except ValueError:
                        pass

                # VR 쿼리
                self._ser.write(b"vr?\n")
                resp = self._ser.readline().decode("utf-8", errors="ignore").strip()
                if resp.startswith("vr:"):
                    try:
                        self.vr_updated.emit(int(resp.split(":")[1]))
                    except ValueError:
                        pass

                self.msleep(300)   # 0.3초 대기 (QThread 전용 sleep)
        except Exception as e:
            self.error_msg.emit(str(e))

    def send(self, cmd: str):
        """메인 스레드에서 호출 — 명령을 시리얼로 전송한다."""
        if self._ser and self._ser.is_open:
            self._ser.write((cmd + "\n").encode())

    def stop(self):
        self._running = False
        self.wait()
        if self._ser and self._ser.is_open:
            self._ser.close()

# ── GUI ──────────────────────────────────────────────
class Dashboard(QWidget):
    def __init__(self):
        super().__init__()
        self.setWindowTitle("TiCLE Lite 대시보드")
        self.resize(380, 320)

        # ─ 센서 표시 영역 ─
        sensor_box = QGroupBox("센서 값")
        sensor_layout = QVBoxLayout(sensor_box)

        # CdS
        cds_row = QHBoxLayout()
        cds_row.addWidget(QLabel("CdS (조도):"))
        self.cds_bar   = QProgressBar()
        self.cds_bar.setRange(0, 65535)
        self.cds_label = QLabel("---")
        cds_row.addWidget(self.cds_bar)
        cds_row.addWidget(self.cds_label)
        sensor_layout.addLayout(cds_row)

        # VR
        vr_row = QHBoxLayout()
        vr_row.addWidget(QLabel("VR (가변저항):"))
        self.vr_bar   = QProgressBar()
        self.vr_bar.setRange(0, 65535)
        self.vr_label = QLabel("---")
        vr_row.addWidget(self.vr_bar)
        vr_row.addWidget(self.vr_label)
        sensor_layout.addLayout(vr_row)

        # ─ LED 제어 영역 ─
        led_box    = QGroupBox("LED 제어")
        led_layout = QVBoxLayout(led_box)

        bright_row = QHBoxLayout()
        bright_row.addWidget(QLabel("밝기:"))
        self.bright_slider = QSlider(Qt.Orientation.Horizontal)
        self.bright_slider.setRange(0, 150)
        self.bright_label  = QLabel("0")
        bright_row.addWidget(self.bright_slider)
        bright_row.addWidget(self.bright_label)
        led_layout.addLayout(bright_row)

        self.bright_slider.valueChanged.connect(self.on_bright_changed)

        # ─ 상태 표시 ─
        self.status_label = QLabel("연결 중...")

        # ─ 전체 레이아웃 ─
        main_layout = QVBoxLayout(self)
        main_layout.addWidget(sensor_box)
        main_layout.addWidget(led_box)
        main_layout.addWidget(self.status_label)

        # ─ 워커 스레드 시작 ─
        self.worker = SensorWorker(PORT, BAUDRATE)
        self.worker.cds_updated.connect(self.update_cds)
        self.worker.vr_updated.connect(self.update_vr)
        self.worker.error_msg.connect(self.show_error)
        self.worker.start()
        self.status_label.setText(f"연결됨: {PORT}")

    def update_cds(self, value):
        self.cds_bar.setValue(value)
        self.cds_label.setText(str(value))

    def update_vr(self, value):
        self.vr_bar.setValue(value)
        self.vr_label.setText(str(value))

    def on_bright_changed(self, value):
        self.bright_label.setText(str(value))
        self.worker.send(f"bright:{value}")

    def show_error(self, msg):
        self.status_label.setText(f"오류: {msg}")

    def closeEvent(self, event):
        self.worker.stop()
        event.accept()

if __name__ == "__main__":
    app = QApplication(sys.argv)
    win = Dashboard()
    win.show()
    sys.exit(app.exec())
```

### 코드의 구조 이해하기

```
Dashboard (QWidget, 메인 스레드)
  ├── QProgressBar (cds_bar)  ← update_cds() 슬롯이 업데이트
  ├── QProgressBar (vr_bar)   ← update_vr() 슬롯이 업데이트
  └── QSlider (bright_slider) ── valueChanged ──→ on_bright_changed()

```

        if self.ser and self.ser.is_open:
            self.ser.close()

## 10. 자주 발생하는 오류

| 증상 | 원인 | 해결 방법 |
|------|------|----------|
| 창이 뜨자마자 닫힘 | `app.exec()` 호출 누락 | `sys.exit(app.exec())` 추가 |
| GUI가 완전히 멈춤 | 메인 스레드에서 `time.sleep()` 또는 블로킹 루프 사용 | `QThread` 또는 `QTimer`로 분리 |
| 크로스 스레드 경고 | 워커 스레드에서 위젯 직접 수정 | `Signal.emit()`으로 메인 스레드에 전달 |
| 창이 보이지 않음 | `win.show()` 호출 누락 | `win.show()` 추가 |
| 시리얼 포트 열기 실패 | 다른 프로그램(Thonny 등)이 포트 점유 | 다른 IDE/터미널의 연결 해제 후 시도 |
| `Signal` import 오류 | PyQt5 `pyqtSignal`을 PySide6에서 사용 | PySide6는 `from PySide6.QtCore import Signal` 사용 |
