# MQTT 통신

## 학습 목표

- Wi-Fi로 인터넷에 연결하고 접속 결과를 확인할 수 있다.
- MQTT 브로커에 접속해 메시지를 발행하고 구독할 수 있다.
- 두 장치가 토픽을 통해 메시지를 주고받는 구조를 설계할 수 있다.

---

## 이 장의 도전 문제

> TiCLE Lite 두 대가 서로 주소를 몰라도 메시지를 주고받을 수 있을까?
> 한 쪽에서 버튼을 누르면 반대쪽 LED에 불이 켜지는 시스템을 만들어 보자.

처음부터 이것을 만들려고 하면 어디서부터 시작해야 할지 막막하다. 아래 미션들을 하나씩 해결하다 보면, 마지막에는 자연스럽게 두 장치가 무선으로 통신하는 시스템이 완성된다.

---

## 사전 지식. 인터넷은 어떻게 동작하는가

스마트폰으로 친구에게 메시지를 보낼 때, 스마트폰이 친구 스마트폰에 직접 연결되는 것이 아니다. 카카오톡 서버를 거쳐 전달된다. MQTT도 같은 방식이다.

```
TiCLE Lite A  ──→  브로커(서버)  ──→  TiCLE Lite B
     (발행자)                              (구독자)
```

**브로커(Broker)**: 메시지를 받아서 맞는 수신자에게 전달하는 서버이다. 우체국과 비슷하다.

**토픽(Topic)**: 메시지가 배달될 주소이다. `/home/lamp/on` 처럼 경로 형태로 쓴다.

**발행(Publish)**: 특정 토픽으로 메시지를 보내는 행위이다.

**구독(Subscribe)**: 특정 토픽에 오는 메시지를 받겠다고 등록하는 행위이다.

핵심은 이것이다. **발행자와 구독자는 서로를 모른다. 토픽만 같으면 메시지가 전달된다.**

## 미션 1. TiCLE Lite를 인터넷에 연결하라

### 문제

Wi-Fi에 연결하고, 연결이 성공했는지 터미널로 확인하라.

### 생각해 보기

TiCLE Lite의 Core 보드에는 Wi-Fi 칩이 내장되어 있다. MicroPython에서는 `network` 모듈로 Wi-Fi를 제어한다.

연결에는 시간이 걸린다. 그래서 "연결이 완료될 때까지 기다리는" 코드가 필요하다. 아래 코드에서 `while not sta.isconnected()` 부분이 그 역할을 한다.

### 코드

파일 이름: `m1_wifi.py`

```python
# m1_wifi.py
import network
import utime

SSID = "여기에 Wi-Fi 이름"
PW   = "여기에 Wi-Fi 비밀번호"

sta = network.WLAN(network.STA_IF)
sta.active(True)
sta.connect(SSID, PW)

print("연결 중...")
while not sta.isconnected():
    utime.sleep_ms(200)
    print(".", end="")

print()
print("연결 성공!")
print("IP:", sta.ifconfig()[0])
```

```sh
replx m1_wifi.py
```

### 관찰하고 실험해 보자

- 연결에 성공하면 IP 주소가 출력된다. 이 숫자는 무엇을 의미하는가?
- Wi-Fi 비밀번호를 일부러 틀리게 입력하면 어떻게 되는가?
- `sta.isconnected()`는 어떤 값을 돌려주는가? `True`와 `False` 중 어떤 것인가?

### 알게 된 것

IP 주소는 인터넷 세계에서 내 장치의 주소이다. 집 주소처럼 이 숫자로 장치를 찾는다. Wi-Fi에 연결된다는 것은 브로커 서버에 접근할 수 있는 네트워크에 진입했다는 뜻이다.

## 미션 2. 메시지를 한 번만 보내 보자

### 문제

Wi-Fi에 연결한 뒤, 공개 브로커에 메시지 하나를 발행하라.

### 생각해 보기

공개 브로커란 누구나 무료로 접속해 테스트할 수 있는 MQTT 서버이다. 이 실습에서는 `test.mosquitto.org`를 사용한다.

메시지를 보내는 흐름은 다음과 같다.

```
1. Wi-Fi 연결
2. 브로커 접속
3. 토픽 결정: "ticle/실습/hello"
4. 메시지 발행: "안녕"
5. 연결 종료
```

토픽 이름을 정할 때 주의할 점이 있다. `test.mosquitto.org`는 전 세계 누구나 접속하는 서버이다. 따라서 **다른 사람과 겹치지 않는 고유한 토픽 이름**을 써야 한다. `ticle/학교이름/이름` 처럼 구체적으로 만드는 것이 좋다.

### 코드

파일 이름: `m2_publish.py`

```python
# m2_publish.py
import network, utime
from umqtt.simple2 import MQTTClient

SSID   = "여기에 Wi-Fi 이름"
PW     = "여기에 Wi-Fi 비밀번호"
BROKER = "test.mosquitto.org"
TOPIC  = b"ticle/my/hello"   # b"..." 는 바이트 문자열이다

# Wi-Fi 연결
sta = network.WLAN(network.STA_IF)
sta.active(True)
sta.connect(SSID, PW)
while not sta.isconnected():
    utime.sleep_ms(200)
print("Wi-Fi OK")

# 브로커 접속 및 메시지 발행
client = MQTTClient(b"pico-sender", BROKER)
client.connect()
client.publish(TOPIC, b"hello from TiCLE Lite")
client.disconnect()
print("메시지 발행 완료")
```

```sh
replx m2_publish.py
```

### 메시지 수신 확인 방법

PC에서 MQTTX(무료 앱)를 실행해 같은 토픽을 구독하면 메시지가 보인다.

1. MQTTX 설치: https://mqttx.app
2. 브로커 주소 `test.mosquitto.org`, 포트 `1883`으로 연결한다.
3. 구독 토픽: `ticle/my/hello` (자신이 쓴 토픽과 동일하게)
4. `replx m2_publish.py` 실행 → MQTTX에 메시지가 나타난다.

### 관찰하고 실험해 보자

- 토픽 이름을 `ticle/my/hello2`로 바꾸고 다시 실행하면 MQTTX에서 어느 토픽에 메시지가 오는가?
- 메시지 내용을 `b"hello from TiCLE Lite"` 대신 숫자로 바꾸려면 어떻게 쓰면 되는가?
- 같은 반 친구와 **같은 토픽**을 구독하면 어떻게 되는가?

### 알게 된 것

`b"..."` 앞의 `b`는 바이트(byte) 문자열을 의미한다. MQTT는 내부적으로 0과 1의 바이트 데이터를 주고받기 때문에 `b`를 붙여야 한다. 토픽과 메시지 모두 바이트 형태로 전달된다.

## 미션 3. 메시지를 받아 보자

### 문제

특정 토픽을 구독하고, 메시지가 도착하면 터미널에 출력하라.

### 생각해 보기

발행(publish)은 한 번 보내고 끝이지만, 구독(subscribe)은 계속 기다려야 한다. 그래서 반복문 안에서 주기적으로 새 메시지가 있는지 확인해야 한다.

새 메시지가 있을 때 자동으로 실행되는 함수를 **콜백(callback)** 이라고 한다. 전화를 걸어 두고 상대방이 받으면 연결되는 것과 비슷하다. 코드를 미리 등록해 두면, 메시지가 오는 순간 그 코드가 실행된다.

### 코드

파일 이름: `m3_subscribe.py`

```python
# m3_subscribe.py
import network, utime
from umqtt.simple2 import MQTTClient

SSID   = "여기에 Wi-Fi 이름"
PW     = "여기에 Wi-Fi 비밀번호"
BROKER = "test.mosquitto.org"
TOPIC  = b"ticle/my/hello"   # 미션 2와 같은 토픽

sta = network.WLAN(network.STA_IF)
sta.active(True)
sta.connect(SSID, PW)
while not sta.isconnected():
    utime.sleep_ms(200)
print("Wi-Fi OK")

# 메시지가 도착했을 때 자동으로 실행될 함수
def on_message(topic, msg, retain, dup):
    print("받은 토픽:", topic)
    print("받은 내용:", msg)
    print("---")

client = MQTTClient(b"pico-receiver", BROKER)
client.set_callback(on_message)   # 콜백 등록
client.connect()
client.subscribe(TOPIC)           # 구독 시작
print("대기 중... (MQTTX에서 메시지를 보내 보자)")

while True:
    client.check_msg()   # 새 메시지 확인
    utime.sleep_ms(100)
```

```sh
replx m3_subscribe.py
```

이 코드를 실행한 뒤, MQTTX에서 `ticle/my/hello` 토픽으로 메시지를 발행하면 터미널에 내용이 출력된다.

### 관찰하고 실험해 보자

- MQTTX에서 메시지를 여러 번 보내면 터미널에 모두 출력되는가?
- `client.check_msg()` 줄을 지우면 어떻게 되는가?
- 다른 토픽(`ticle/my/other`)으로 메시지를 보내면 출력되는가?

### 알게 된 것

`check_msg()`는 브로커에 새 메시지가 있으면 콜백을 호출하고, 없으면 그냥 넘어간다. 반복문에서 계속 호출해야 메시지를 놓치지 않는다. 이처럼 "정기적으로 확인하는" 방식을 **폴링(polling)** 이라고 한다.

## 미션 4. 받은 메시지로 LED를 켜라

### 문제

PC에서 `"on"` 메시지를 보내면 LED가 켜지고, `"off"` 메시지를 보내면 꺼지게 만들어라.

### 생각해 보기

미션 3까지는 메시지를 받아 출력만 했다. 이제 받은 내용을 **조건에 따라 다르게 처리**해 보자. 콜백 함수 안에서 `msg` 값을 검사하면 된다.

수신된 `msg`는 바이트 문자열이다. `b"on"` 처럼 비교해야 한다. 일반 문자열 `"on"`과는 다르다.

### 코드

파일 이름: `m4_led_control.py`

```python
# m4_led_control.py
import network, utime
from machine import Pin
from umqtt.simple2 import MQTTClient

SSID   = "여기에 Wi-Fi 이름"
PW     = "여기에 Wi-Fi 비밀번호"
BROKER = "test.mosquitto.org"
TOPIC  = b"ticle/my/led"

sta = network.WLAN(network.STA_IF)
sta.active(True)
sta.connect(SSID, PW)
while not sta.isconnected():
    utime.sleep_ms(200)
print("Wi-Fi OK")

led = Pin('LED', Pin.OUT)

def on_message(topic, msg, retain, dup):
    print("명령 수신:", msg)
    if msg == b"on":
        led.on()
        print("LED 켜짐")
    elif msg == b"off":
        led.off()
        print("LED 꺼짐")
    else:
        print("알 수 없는 명령:", msg)

client = MQTTClient(b"pico-led", BROKER)
client.set_callback(on_message)
client.connect()
client.subscribe(TOPIC)
print("LED 제어 대기 중...")

while True:
    client.check_msg()
    utime.sleep_ms(100)
```

```sh
replx m4_led_control.py
```

MQTTX에서 `ticle/my/led` 토픽으로 `on` 또는 `off`를 보내면 LED가 반응한다.

### 관찰하고 실험해 보자

- `"ON"`, `"ON "`, `" on"` (대문자 또는 공백 포함)을 보내면 어떻게 되는가?
- `"on"`과 `"off"` 외에 `"blink"` 명령도 처리하도록 코드를 추가해 보자.
- 명령어를 영어 대신 숫자 `b"1"`, `b"0"`으로 바꾸어도 동작하도록 수정해 보자.

### 알게 된 것

`if msg == b"on"` 조건에서 `b"on"`과 `"on"`은 다르다. 전자는 바이트 문자열, 후자는 유니코드 문자열이다. MQTT로 수신한 메시지는 항상 바이트 문자열이므로 `b""` 형태로 비교해야 한다.

## 미션 5. 이번엔 TiCLE Lite가 먼저 말을 걸게 하자

### 문제

버튼을 누를 때마다 눌린 횟수를 MQTT로 발행하라.

### 생각해 보기

지금까지는 외부에서 명령을 받아 반응했다. 이번엔 반대로 TiCLE Lite가 스스로 정보를 내보내는 역할을 한다. **센서 데이터를 주기적으로 외부에 알리는 것**이 IoT의 핵심 기능 중 하나이다.

버튼을 눌렀는지 감지하려면 "이전 상태"와 "현재 상태"를 비교해야 한다. 이전에 눌리지 않았다가 지금 눌렸을 때만 횟수를 올려야 한다.

```
이전: 0(안 눌림), 현재: 1(눌림)  →  이벤트 발생!
이전: 1(눌림),   현재: 1(눌림)  →  계속 눌리는 중, 무시
이전: 1(눌림),   현재: 0(안 눌림) →  손을 뗌, 무시
```

### 코드

파일 이름: `m5_button_pub.py`

```python
# m5_button_pub.py
import network, utime
from machine import Pin
from umqtt.simple2 import MQTTClient

SSID   = "여기에 Wi-Fi 이름"
PW     = "여기에 Wi-Fi 비밀번호"
BROKER = "test.mosquitto.org"
TOPIC  = b"ticle/my/button"

sta = network.WLAN(network.STA_IF)
sta.active(True)
sta.connect(SSID, PW)
while not sta.isconnected():
    utime.sleep_ms(200)
print("Wi-Fi OK")

btn      = Pin(16, Pin.IN)   # UP 버튼
prev_val = 0
count    = 0

client = MQTTClient(b"pico-button", BROKER)
client.connect()
print("버튼을 눌러 보자!")

while True:
    cur_val = btn.value()
    if cur_val == 1 and prev_val == 0:   # 새로 눌린 순간
        count += 1
        msg = str(count).encode()        # 숫자를 바이트로 변환
        client.publish(TOPIC, msg)
        print("발행:", msg)
    prev_val = cur_val
    utime.sleep_ms(20)
```

```sh
replx m5_button_pub.py
```

### 관찰하고 실험해 보자

- MQTTX에서 `ticle/my/button`을 구독한 뒤 버튼을 눌러 보자. 메시지가 도착하는가?
- 버튼을 꾹 누르고 있으면 숫자가 계속 올라가는가, 아니면 한 번만 올라가는가? 왜 그 결과가 나오는가?
- `str(count).encode()` 대신 `f"버튼 {count}번 눌림".encode()`로 바꾸면 어떻게 출력되는가?

### 알게 된 것

`str(count).encode()`는 정수(예: `5`)를 문자열(`"5"`)로 바꾼 뒤, 다시 바이트(`b"5"`)로 변환한다. MQTT는 문자열도 숫자도 아닌 바이트만 주고받기 때문에 이 변환이 항상 필요하다.

## 최종 프로젝트. 두 TiCLE Lite가 대화하게 만들어라

### 설계

지금까지 배운 내용을 합쳐서 다음을 만든다.

- **보드 A**: UP 버튼을 누르면 `"on"`, DOWN 버튼을 누르면 `"off"`를 발행한다.
- **보드 B**: 같은 토픽을 구독하고 `"on"` 수신 시 LED 켜기, `"off"` 수신 시 LED 끄기를 실행한다.
- 보드가 한 대뿐이라면, MQTTX가 보드 A 역할을 대신한다.

| 장치 | 역할 | 토픽 |
|------|------|------|
| 보드 A (또는 MQTTX) | 입력 → 발행 | `ticle/my/lamp` 로 발행 |
| 보드 B | 구독 → LED 제어 | `ticle/my/lamp` 구독 |

### 최종 코드

파일 이름: `mqtt_project.py`

```python
# mqtt_project.py
# 발행과 구독을 동시에 처리하는 TiCLE Lite MQTT 프로젝트
import network, utime
from machine import Pin
from umqtt.simple2 import MQTTClient

SSID      = "여기에 Wi-Fi 이름"
PW        = "여기에 Wi-Fi 비밀번호"
BROKER    = "test.mosquitto.org"
TOPIC_PUB = b"ticle/my/lamp"   # 내가 발행하는 토픽
TOPIC_SUB = b"ticle/my/lamp"   # 내가 구독하는 토픽

# Wi-Fi 연결
sta = network.WLAN(network.STA_IF)
sta.active(True)
sta.connect(SSID, PW)
while not sta.isconnected():
    utime.sleep_ms(200)
print("Wi-Fi OK  IP:", sta.ifconfig()[0])

btn_up   = Pin(16, Pin.IN)   # UP 버튼 → "on" 발행
btn_down = Pin(18, Pin.IN)   # DOWN 버튼 → "off" 발행
led      = Pin('LED', Pin.OUT)

prev_up   = 0
prev_down = 0

def on_message(topic, msg, retain, dup):
    if msg == b"on":
        led.on()
        print("LED ON")
    elif msg == b"off":
        led.off()
        print("LED OFF")

client = MQTTClient(b"pico-main", BROKER)
client.set_callback(on_message)
client.connect()
client.subscribe(TOPIC_SUB)
print("준비 완료. UP=켜기, DOWN=끄기")

while True:
    # 구독 메시지 확인
    client.check_msg()

    # 버튼 입력 처리
    cur_up   = btn_up.value()
    cur_down = btn_down.value()

    if cur_up == 1 and prev_up == 0:
        client.publish(TOPIC_PUB, b"on")
        print("'on' 발행")

    if cur_down == 1 and prev_down == 0:
        client.publish(TOPIC_PUB, b"off")
        print("'off' 발행")

    prev_up   = cur_up
    prev_down = cur_down
    utime.sleep_ms(20)
```
