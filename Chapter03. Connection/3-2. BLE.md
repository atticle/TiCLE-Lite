# BLE 통신

## 학습 목표

- BLE 광고(Advertising) 패킷을 만들어 장치가 발견되도록 만들 수 있다.
- GATT 서비스와 특성(Characteristic)을 만들어 스마트폰과 데이터를 주고받을 수 있다.
- 수신한 문자열 명령으로 LED를 제어하는 BLE 장치를 완성할 수 있다.

---

## 이 장의 도전 문제

> 인터넷이 없어도, Wi-Fi 비밀번호를 몰라도, 스마트폰 하나로 TiCLE Lite를 제어할 수 있을까?
> 스마트폰 앱에서 `"on"`을 보내면 LED가 켜지는 BLE 장치를 만들어 보자.

BLE는 **Bluetooth Low Energy**의 약자이다. 일반 블루투스보다 전력 소모가 적고, 스마트폰 앱으로 직접 제어하기에 적합하다. 이어폰, 심박 센서, 스마트 잠금장치 등이 BLE를 사용한다.

---

## 사전 지식. BLE는 어떻게 동작하는가

BLE 통신에는 두 가지 역할이 있다.

| 역할 | 설명 | 예시 |
|------|------|------|
| Peripheral (주변 장치) | 광고를 내보내고, 연결을 기다린다 | TiCLE Lite, 이어폰, 심박계 |
| Central (중앙 장치) | 주변 장치를 스캔하고, 연결을 시작한다 | 스마트폰, 태블릿 |

연결이 이루어지면 데이터 교환은 **GATT** 구조를 통해 이루어진다.

```
GATT 구조
└── Service (서비스: 기능 묶음)
    └── Characteristic (특성: 데이터 항목 하나)
        ├── Read  (중앙이 읽어 감)
        ├── Write (중앙이 써 줌)
        └── Notify (주변이 자동으로 알림)
```

서비스는 폴더, 특성은 파일 하나라고 생각하면 된다. 각각은 UUID라는 고유한 번호로 구분된다.

---

## 준비. 스마트폰 앱 설치

스마트폰에 **nRF Connect** (Nordic Semiconductor, 무료) 앱을 설치한다. 이 앱으로 BLE 장치를 스캔하고, 데이터를 읽고 쓸 수 있다.

## 미션 1. 스마트폰이 내 장치를 찾게 하라

### 문제

TiCLE Lite가 `"TiCLE-BLE"` 라는 이름으로 BLE 광고를 내보내도록 만들어라. 스마트폰에서 장치가 보여야 한다.

### 생각해 보기

BLE에서 Peripheral은 주기적으로 자신의 존재를 알리는 신호를 보낸다. 이것을 **광고(Advertising)** 라고 한다. 마치 가게 간판이 "여기 가게 있어요"라고 알리는 것과 같다.

광고 패킷에는 장치 이름, 지원하는 서비스 UUID 등의 정보가 담긴다.

### 코드

파일 이름: `m1_advertise.py`

```python
# m1_advertise.py
import bluetooth
import struct
import utime

ble = bluetooth.BLE()
ble.active(True)

# 광고 패킷 조립
# 형식: [길이, 타입, 데이터, ...]
name      = "TiCLE-BLE"
name_enc  = name.encode()

# 0x01: Flags (일반 발견 가능 + BR/EDR 미지원)
# 0x09: Complete Local Name
adv_data = (
    bytes([2, 0x01, 0x06]) +
    bytes([1 + len(name_enc), 0x09]) + name_enc
)

ble.gap_advertise(100_000, adv_data)   # 100ms 간격으로 광고
print("광고 시작:", name)
print("스마트폰의 nRF Connect에서 스캔해 보자")

# 광고를 계속 유지하며 대기
while True:
    utime.sleep_ms(1000)
    print("광고 중...")
```

```sh
replx m1_advertise.py
```

### 관찰하고 실험해 보자

- nRF Connect 앱에서 스캔을 시작하면 `TiCLE-BLE`가 목록에 나타나는가?
- 프로그램을 종료(`Ctrl+C`)하면 장치가 목록에서 사라지는가?
- 장치 이름을 자신의 이름으로 바꾸어도 잘 나타나는가?

### 알게 된 것

광고 패킷은 정해진 형식의 바이트 배열이다. `0x01`은 Flags(동작 모드), `0x09`는 장치 이름을 나타내는 타입 코드이다. 이 코드들은 블루투스 표준에서 정해진 약속이다.

## 미션 2. 스마트폰이 데이터를 쓸 수 있는 항목을 만들어라

### 문제

스마트폰에서 문자열을 쓸 수 있는 Write 특성을 만들고, 쓰인 내용을 터미널에 출력하라.

### 생각해 보기

광고만으로는 데이터를 주고받을 수 없다. 연결 후 데이터를 교환하려면 GATT 서비스와 특성을 만들어야 한다.

UUID는 서비스와 특성을 구분하는 고유 번호이다. 표준 UUID(16비트)도 있고, 커스텀 UUID(128비트)도 만들 수 있다. 아래 예제에서는 커스텀 UUID를 사용한다.

```
서비스 UUID: 12345678-1234-1234-1234-123456789abc
특성 UUID:   87654321-4321-4321-4321-cba987654321
```

### 코드

파일 이름: `m2_write.py`

```python
# m2_write.py
import bluetooth
import struct
import utime
from micropython import const

_FLAG_WRITE = const(0x0008)

# UUID 정의
_SVC_UUID  = bluetooth.UUID("12345678-1234-1234-1234-123456789abc")
_CHAR_UUID = bluetooth.UUID("87654321-4321-4321-4321-cba987654321")

# 서비스 등록
_SERVICE = (
    _SVC_UUID,
    [(_CHAR_UUID, _FLAG_WRITE)],
)

ble     = bluetooth.BLE()
ble.active(True)

# 서비스 등록 후 핸들 받기
((char_handle,),) = ble.gatts_register_services((_SERVICE,))

received_data = []

def ble_irq(event, data):
    if event == 3:   # _IRQ_GATTS_WRITE (값이 쓰임)
        buf   = ble.gatts_read(char_handle)
        value = buf.decode().strip()
        print("수신:", value)
        received_data.append(value)

ble.irq(ble_irq)

# 광고 패킷
name     = "TiCLE-BLE"
name_enc = name.encode()
adv_data = (
    bytes([2, 0x01, 0x06]) +
    bytes([1 + len(name_enc), 0x09]) + name_enc
)
ble.gap_advertise(100_000, adv_data)
print("준비 완료 — nRF Connect로 연결해 Write 특성에 문자열을 써 보자")

while True:
    utime.sleep_ms(100)
```

```sh
replx m2_write.py
```

### nRF Connect로 테스트하는 방법

1. 앱에서 `TiCLE-BLE` 장치에 연결한다.
2. 서비스 목록에서 `12345678...` UUID를 찾는다.
3. Write 특성 옆의 위쪽 화살표(↑) 버튼을 누른다.
4. `UTF-8` 형식으로 `hello` 를 입력하고 전송한다.
5. 터미널에 `수신: hello`가 출력됐는지 확인한다.

### 관찰하고 실험해 보자

- 숫자 `123`을 보내면 터미널에 어떻게 출력되는가?
- 한글 `안녕`을 보내면 정상 출력되는가?
- `received_data` 리스트에 받은 데이터가 쌓이고 있는가?

### 알게 된 것

`ble.irq(ble_irq)`는 BLE 이벤트가 발생할 때마다 `ble_irq` 함수를 호출하도록 등록하는 것이다. 이벤트 번호 `3`은 "누군가 Write 특성에 값을 썼다"는 신호이다. MQTT의 콜백과 같은 원리이다.

## 미션 3. 받은 명령으로 LED를 켜고 꺼라

### 문제

Write 특성에 `"on"`이 수신되면 LED를 켜고, `"off"`가 수신되면 LED를 끄라.

### 생각해 보기

미션 2에서 문자열을 수신하는 방법을 배웠다. 이제 수신된 값을 조건에 따라 처리하면 된다. MQTT 미션 4에서 했던 것과 구조가 같다. **인터페이스(BLE / MQTT)는 달라도 "받아서 판단해 반응한다"는 패턴은 동일하다.**

### 코드

파일 이름: `m3_led_ble.py`

```python
# m3_led_ble.py
import bluetooth
import utime
from machine import Pin
from micropython import const

_FLAG_WRITE = const(0x0008)

_SVC_UUID  = bluetooth.UUID("12345678-1234-1234-1234-123456789abc")
_CHAR_UUID = bluetooth.UUID("87654321-4321-4321-4321-cba987654321")

_SERVICE = (
    _SVC_UUID,
    [(_CHAR_UUID, _FLAG_WRITE)],
)

ble = bluetooth.BLE()
ble.active(True)
((char_handle,),) = ble.gatts_register_services((_SERVICE,))

led = Pin('LED', Pin.OUT)

def ble_irq(event, data):
    if event == 3:
        cmd = ble.gatts_read(char_handle).decode().strip()
        print("명령:", cmd)
        if cmd == "on":
            led.on()
            print("LED 켜짐")
        elif cmd == "off":
            led.off()
            print("LED 꺼짐")
        else:
            print("알 수 없는 명령")

ble.irq(ble_irq)

name     = "TiCLE-BLE"
name_enc = name.encode()
adv_data = (
    bytes([2, 0x01, 0x06]) +
    bytes([1 + len(name_enc), 0x09]) + name_enc
)
ble.gap_advertise(100_000, adv_data)
print("BLE LED 제어 준비 완료")

while True:
    utime.sleep_ms(100)
```

```sh
replx m3_led_ble.py
```

### 관찰하고 실험해 보자

- nRF Connect에서 `on`과 `off`를 번갈아 보내면 LED가 반응하는가?
- MQTT를 이용할 때와 이 방식의 차이점은 무엇인가? (인터넷 필요 여부, 거리 제한 등)
- `"blink"` 명령도 처리하게 만들려면 코드 어디를 어떻게 바꾸면 되는가?

### 알게 된 것

BLE와 MQTT는 모두 "명령을 받아 반응한다"는 구조이지만, 통신 경로가 다르다. BLE는 인터넷 없이 근거리에서 직접 연결되고, MQTT는 인터넷을 통한 브로커를 거친다. 사용하는 환경에 따라 적합한 방식을 선택해야 한다.

## 미션 4. TiCLE Lite가 스마트폰에 데이터를 보내게 하라

### 문제

1초마다 현재 CdS 센서값을 Notify로 스마트폰에 전송하라.

### 생각해 보기

지금까지는 스마트폰 → TiCLE Lite 방향(Write)만 해봤다. 이번엔 반대 방향이다. TiCLE Lite가 스마트폰에 알림(Notify)을 보내면, 스마트폰은 받은 값을 화면에 표시한다. 심박계가 심박수를 앱에 계속 전송하는 것과 같은 원리이다.

Notify를 사용하려면 특성 플래그에 `_FLAG_NOTIFY`를 추가해야 한다.

### 코드

파일 이름: `m4_notify.py`

```python
# m4_notify.py
import bluetooth
import utime
from machine import Pin, ADC
from micropython import const

_FLAG_NOTIFY = const(0x0010)

_SVC_UUID  = bluetooth.UUID("12345678-1234-1234-1234-123456789abc")
_CHAR_UUID = bluetooth.UUID("AAAABBBB-CCCC-DDDD-EEEE-FFFFFFFFFFFF")

_SERVICE = (
    _SVC_UUID,
    [(_CHAR_UUID, _FLAG_NOTIFY)],
)

ble = bluetooth.BLE()
ble.active(True)
((char_handle,),) = ble.gatts_register_services((_SERVICE,))

cds   = ADC(Pin(28))
conn  = None   # 현재 연결된 Central 핸들

def ble_irq(event, data):
    global conn
    if event == 1:    # _IRQ_CENTRAL_CONNECT
        conn = data[0]
        print("연결됨! 핸들:", conn)
    elif event == 2:  # _IRQ_CENTRAL_DISCONNECT
        conn = None
        print("연결 끊김, 광고 재시작")
        ble.gap_advertise(100_000, adv_data)

ble.irq(ble_irq)

name     = "TiCLE-Sensor"
name_enc = name.encode()
adv_data = (
    bytes([2, 0x01, 0x06]) +
    bytes([1 + len(name_enc), 0x09]) + name_enc
)
ble.gap_advertise(100_000, adv_data)
print("광고 시작 — 연결 후 Notify 수신 대기")

while True:
    if conn is not None:
        val     = cds.read_u16()
        payload = str(val).encode()
        ble.gatts_notify(conn, char_handle, payload)
        print("전송:", val)
    utime.sleep_ms(1000)
```

```sh
replx m4_notify.py
```

### 관찰하고 실험해 보자

- nRF Connect에서 Notify 특성 구독 후 CdS 센서를 손으로 가리면 숫자가 바뀌는가?
- 연결이 끊기면 자동으로 광고를 다시 시작하는가?
- Notify 데이터를 문자열이 아닌 정수 바이트로 효율적으로 보내려면 어떻게 하면 되는가?

## 최종 프로젝트. BLE로 제어하고, 센서 데이터도 되돌려 받아라

### 설계

| 특성 | 플래그 | 역할 |
|------|--------|------|
| Write 특성 | WRITE | 스마트폰 → TiCLE Lite 명령 수신 |
| Notify 특성 | NOTIFY | TiCLE Lite → 스마트폰 센서값 전송 |

Write로 `"led_on"`, `"led_off"` 명령을 받고, 동시에 CdS 값을 Notify로 전송한다.

### 최종 코드

파일 이름: `ble_project.py`

```python
# ble_project.py
import bluetooth
import utime
from machine import Pin, ADC
from micropython import const

_FLAG_WRITE  = const(0x0008)
_FLAG_NOTIFY = const(0x0010)

_SVC_UUID    = bluetooth.UUID("12345678-1234-1234-1234-123456789abc")
_CMD_UUID    = bluetooth.UUID("87654321-4321-4321-4321-cba987654321")  # Write
_SENSOR_UUID = bluetooth.UUID("AAAABBBB-CCCC-DDDD-EEEE-FFFFFFFFFFFF")  # Notify

_SERVICE = (
    _SVC_UUID,
    [
        (_CMD_UUID,    _FLAG_WRITE),
        (_SENSOR_UUID, _FLAG_NOTIFY),
    ],
)

ble = bluetooth.BLE()
ble.active(True)
((cmd_handle, sensor_handle),) = ble.gatts_register_services((_SERVICE,))

led  = Pin('LED', Pin.OUT)
cds  = ADC(Pin(28))
conn = None

name     = "TiCLE-BLE"
name_enc = name.encode()
adv_data = (
    bytes([2, 0x01, 0x06]) +
    bytes([1 + len(name_enc), 0x09]) + name_enc
)

def ble_irq(event, data):
    global conn
    if event == 1:   # 연결
        conn = data[0]
        print("연결됨")
    elif event == 2: # 끊김
        conn = None
        print("끊김, 광고 재시작")
        ble.gap_advertise(100_000, adv_data)
    elif event == 3: # Write 수신
        cmd = ble.gatts_read(cmd_handle).decode().strip()
        print("명령:", cmd)
        if cmd == "led_on":
            led.on()
        elif cmd == "led_off":
            led.off()

ble.irq(ble_irq)
ble.gap_advertise(100_000, adv_data)
print("BLE 프로젝트 준비 완료")

while True:
    if conn is not None:
        val = cds.read_u16()
        ble.gatts_notify(conn, sensor_handle, str(val).encode())
    utime.sleep_ms(500)
```

```sh
replx ble_project.py
```

## MQTT vs BLE 비교

| 항목 | MQTT | BLE |
|------|------|-----|
| 필요한 인프라 | Wi-Fi + 인터넷 + 브로커 | 스마트폰만 있으면 됨 |
| 통달 거리 | 인터넷 범위 (전 세계) | 약 10~50m |
| 동시 장치 수 | 수천 대 | 보통 1대와 연결 |
| 전력 소모 | 상대적으로 높음 | 매우 낮음 |
| 적합한 사용 예 | 원격 모니터링, 스마트홈 | 웨어러블, 근거리 제어 |
