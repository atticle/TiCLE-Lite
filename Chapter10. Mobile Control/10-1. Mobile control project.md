# 모바일 앱으로 제어하기

## 학습 목표

- Blynk IoT 플랫폼의 가상 핀(Virtual Pin) 구조를 이해하고 설명할 수 있다.
- TiCLE Lite에서 센서 데이터를 Blynk 클라우드로 전송할 수 있다.
- 스마트폰 앱의 슬라이더로 LED 밝기를 실시간 제어할 수 있다.

---

## 이 장의 도전 문제

> 코딩 없이 만든 스마트폰 앱으로 TiCLE Lite를 제어할 수 있을까?
> 앱의 슬라이더를 움직이면 LED 밝기가 바뀌고, LED 화면의 센서 데이터가 앱에 표시되게 만들어 보자.

---

## Blynk란 무엇인가

Blynk는 앱 개발 없이 모바일 대시보드를 만들 수 있는 IoT 플랫폼이다. 슬라이더, 버튼, 게이지, 그래프 같은 위젯을 드래그 앤 드롭으로 배치하면 된다. 내부적으로 MQTT를 사용해 데이터를 주고받는다.

```
스마트폰 앱 (Blynk)  ↔  Blynk 클라우드  ↔  TiCLE Lite
    슬라이더/게이지         MQTT 중계          센서/LED
```

---

## 준비. Blynk 계정과 앱 설정

1. https://blynk.io 에서 무료 계정을 만든다.
2. **Blynk app** (iOS / Android)을 설치한다.
3. 새 프로젝트를 만들고 장치 유형을 **Raspberry Pi Pico W** 로 선택한다.
4. **Auth Token** (인증 토큰)을 복사해 둔다. 아래 코드에서 사용한다.
5. 다음 위젯을 추가한다.

| 위젯 | 가상 핀 | 역할 |
|------|---------|------|
| Slider | V0 | LED 밝기 조절 (0~255) |
| Gauge | V1 | CdS 센서값 표시 |

## 미션 1. Blynk에 연결하고 하트비트를 보내라

### 문제

Wi-Fi와 Blynk 서버에 연결한 뒤, 연결 상태를 1초마다 터미널에 출력하라.

### 코드

파일 이름: `m1_blynk_connect.py`

```python
# m1_blynk_connect.py
from blynk_mqtt import BlynkMQTT
import network, utime

SSID  = "여기에 Wi-Fi 이름"
PW    = "여기에 Wi-Fi 비밀번호"
TOKEN = "여기에 Blynk Auth Token"

sta = network.WLAN(network.STA_IF)
sta.active(True)
sta.connect(SSID, PW)
while not sta.isconnected():
    utime.sleep_ms(200)
print("Wi-Fi OK")

blynk = BlynkMQTT(TOKEN)
blynk.connect()
print("Blynk 연결됨")

while True:
    blynk.run()   # 내부 처리 (연결 유지, 메시지 처리)
    utime.sleep_ms(100)
```

## 미션 2. CdS 센서값을 앱의 게이지로 실시간 표시하라

### 문제

CdS 값을 2초마다 읽어 Blynk 가상 핀 V1으로 전송하라. 앱의 게이지 위젯에 숫자가 나타나야 한다.

### 생각해 보기

Blynk에서 데이터를 전송하는 방법은 `blynk.virtual_write(핀번호, 값)` 이다. 가상 핀은 물리 핀과 달리 앱 위젯과 연결된 소프트웨어 채널이다.

### 코드

파일 이름: `m2_sensor_display.py`

```python
# m2_sensor_display.py
from blynk_mqtt import BlynkMQTT
from machine import Pin, ADC
import network, utime

SSID  = "여기에 Wi-Fi 이름"
PW    = "여기에 Wi-Fi 비밀번호"
TOKEN = "여기에 Blynk Auth Token"

sta = network.WLAN(network.STA_IF)
sta.active(True)
sta.connect(SSID, PW)
while not sta.isconnected():
    utime.sleep_ms(200)

cds   = ADC(Pin(28))
blynk = BlynkMQTT(TOKEN)
blynk.connect()

last_send = utime.ticks_ms()

while True:
    blynk.run()

    if utime.ticks_diff(utime.ticks_ms(), last_send) >= 2000:
        val = cds.read_u16()
        blynk.virtual_write(1, val)   # V1 게이지에 전송
        print("CdS →  V1:", val)
        last_send = utime.ticks_ms()

    utime.sleep_ms(50)
```

```sh
replx m2_sensor_display.py
```

## 미션 3. 앱의 슬라이더로 LED 밝기를 제어하라

### 문제

앱 V0 슬라이더를 움직이면 그 값을 받아 LED 매트릭스 전체 밝기를 조절하라.

### 생각해 보기

앱에서 슬라이더를 움직이면 Blynk가 `virtual_pin_handler`를 호출한다. 핀 번호와 값이 인자로 전달된다.

슬라이더 범위가 0~255이므로, 이 값으로 LED 색상을 직접 설정하면 된다. 예를 들어 흰색의 밝기를 조절하려면 `(val, val, val)` 을 사용한다.

### 코드

파일 이름: `m3_slider_brightness.py`

```python
# m3_slider_brightness.py
from blynk_mqtt import BlynkMQTT
from ws2812 import WS2812Matrix
import network, utime

SSID  = "여기에 Wi-Fi 이름"
PW    = "여기에 Wi-Fi 비밀번호"
TOKEN = "여기에 Blynk Auth Token"

sta = network.WLAN(network.STA_IF)
sta.active(True)
sta.connect(SSID, PW)
while not sta.isconnected():
    utime.sleep_ms(200)

m     = WS2812Matrix([(0, 1)])
blynk = BlynkMQTT(TOKEN)

brightness = [0]      # 리스트로 감싸야 내부 함수에서 수정 가능

def on_v0(value):
    """V0 슬라이더 값이 변하면 호출된다."""
    val          = int(float(value))
    val          = min(max(val, 0), 150)   # 최대 150으로 제한
    brightness[0] = val
    for x in range(16):
        for y in range(16):
            m[x, y].value = (val, val, val)
    m.update()
    print("밝기:", val)

blynk.on_virtual_write(0, on_v0)   # V0 핸들러 등록
blynk.connect()

while True:
    blynk.run()
    utime.sleep_ms(50)
```


## 최종 프로젝트. 양방향 모바일 대시보드

### 설계

| 앱 위젯 | 가상 핀 | 방향 | 기능 |
|---------|---------|------|------|
| Slider | V0 | 앱 → 보드 | LED 밝기 |
| Gauge | V1 | 보드 → 앱 | CdS 센서값 |
| Gauge | V2 | 보드 → 앱 | VR 센서값 |

### 최종 코드

파일 이름: `mobile_project.py`

```python
# mobile_project.py
from blynk_mqtt import BlynkMQTT
from ws2812 import WS2812Matrix
from machine import Pin, ADC
import network, utime

SSID  = "여기에 Wi-Fi 이름"
PW    = "여기에 Wi-Fi 비밀번호"
TOKEN = "여기에 Blynk Auth Token"

sta = network.WLAN(network.STA_IF)
sta.active(True)
sta.connect(SSID, PW)
while not sta.isconnected():
    utime.sleep_ms(200)
print("Wi-Fi OK")

m   = WS2812Matrix([(0, 1)])
cds = ADC(Pin(28))
vr  = ADC(Pin(27))

blynk     = BlynkMQTT(TOKEN)
led_val   = [0]
last_send = utime.ticks_ms()

def on_v0(value):
    val       = min(int(float(value)), 150)
    led_val[0] = val
    for x in range(16):
        for y in range(16):
            m[x, y].value = (val, val, val)
    m.update()

blynk.on_virtual_write(0, on_v0)
blynk.connect()
print("대시보드 시작")

while True:
    blynk.run()

    if utime.ticks_diff(utime.ticks_ms(), last_send) >= 1000:
        blynk.virtual_write(1, cds.read_u16())
        blynk.virtual_write(2, vr.read_u16())
        last_send = utime.ticks_ms()

    utime.sleep_ms(50)
```

```sh
replx mobile_project.py
```
