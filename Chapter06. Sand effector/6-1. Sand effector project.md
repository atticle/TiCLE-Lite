# 모래 효과

## 학습 목표

- IMU 센서에서 기울기를 읽어 중력 방향을 결정할 수 있다.
- 좌표 기반으로 입자 하나를 이동시키는 규칙을 설계할 수 있다.
- 입자가 쌓이는 충돌 규칙을 추가해 여러 입자가 모래처럼 흐르게 만들 수 있다.

---

## 이 장의 도전 문제

> 보드를 기울이면 LED 화면 안의 점들이 실제 모래처럼 흘러내리게 만들 수 있을까?

이 장의 핵심은 센서를 읽는 것이 아니다. **"입자가 어떤 규칙으로 움직여야 모래처럼 보이는가"** 를 설계하는 것이다. 물리 법칙을 완벽히 계산하는 것이 아니라, 보는 사람이 "모래 같다"고 느끼게 만드는 단순한 규칙이 무엇인지 찾는 것이다.

---

## 사전 지식. IMU 센서란 무엇인가

IMU(Inertial Measurement Unit)는 장치의 기울기와 가속도를 측정하는 센서이다. TiCLE Lite에는 MPU6050이 내장되어 있다.

```python
from mpu6050 import MPU6050
from machine import I2C, Pin

imu = MPU6050(I2C(0, scl=Pin(13), sda=Pin(12), freq=400000))
ax, ay, az = imu.accel   # x, y, z 축 가속도 (단위: g)
```

보드를 수평으로 놓으면 `az ≈ 1.0`, `ax ≈ 0`, `ay ≈ 0` 이다.

보드를 오른쪽으로 기울이면 `ax`가 커지고, 앞으로 기울이면 `ay`가 변한다. 이 값을 이용해 "중력이 당기는 방향"을 결정한다.

---

## 미션 1. 기울기를 숫자로 읽어라

### 문제

0.1초마다 IMU의 가속도 값을 읽고, x축과 y축의 가속도를 출력하라.

### 코드

파일 이름: `m1_imu_read.py`

```python
# m1_imu_read.py
from mpu6050 import MPU6050
from machine import I2C, Pin
import utime

imu = MPU6050(I2C(0, scl=Pin(13), sda=Pin(12), freq=400000))

while True:
    ax, ay, az = imu.accel
    print(f"ax={ax:6.2f}  ay={ay:6.2f}  az={az:6.2f}")
    utime.sleep_ms(100)
```

```sh
replx m1_imu_read.py
```

### 관찰하고 실험해 보자

- 보드를 수평으로 놓으면 세 값이 얼마인가?
- 보드를 오른쪽으로 기울이면 어떤 값이 바뀌는가?
- 보드를 앞뒤로 기울이면 어떤 값이 바뀌는가?
- `ax`가 양수이면 어느 방향으로 기울어진 것인가?

### 알게 된 것

가속도계는 중력이 각 축 방향으로 얼마나 작용하는지 측정한다. 보드를 기울이면 중력 벡터가 각 축에 분배되어 숫자로 나타난다. 이 값의 부호와 크기로 기울어진 방향과 각도를 알 수 있다.

## 미션 2. 기울기를 방향으로 바꿔라

### 문제

가속도 값을 읽어 가장 많이 기울어진 방향을 `"UP"`, `"DOWN"`, `"LEFT"`, `"RIGHT"` 중 하나로 출력하라.

### 생각해 보기

가속도 값은 연속적인 숫자이다. 하지만 우리가 필요한 것은 "어느 방향으로 기울었나?"라는 이산적인 판단이다. 가장 간단한 방법은 네 방향 중 절댓값이 가장 큰 축과 부호를 보는 것이다.

```
|ax| > |ay|:  ax > 0 → RIGHT, ax < 0 → LEFT
|ay| >= |ax|: ay > 0 → DOWN,  ay < 0 → UP
```

### 코드

파일 이름: `m2_direction.py`

```python
# m2_direction.py
from mpu6050 import MPU6050
from machine import I2C, Pin
import utime

imu = MPU6050(I2C(0, scl=Pin(13), sda=Pin(12), freq=400000))

def get_direction(ax, ay):
    if abs(ax) > abs(ay):
        return "RIGHT" if ax > 0 else "LEFT"
    else:
        return "DOWN" if ay > 0 else "UP"

while True:
    ax, ay, _ = imu.accel
    direction = get_direction(ax, ay)
    print(f"ax={ax:5.2f}  ay={ay:5.2f}  →  {direction}")
    utime.sleep_ms(100)
```

```sh
replx m2_direction.py
```

### 관찰하고 실험해 보자

- 보드를 천천히 기울이면 방향이 예상대로 나오는가?
- 보드를 대각선으로 기울이면 출력이 안정적인가, 방향이 왔다 갔다 하는가?
- `abs(ax) > abs(ay)` 조건 대신 임계값을 추가하면(`abs(ax) > 0.3`) 더 안정적이 되는가?

### 알게 된 것

센서 값을 그대로 쓰는 것과 판단을 위해 이산화(discretize)하는 것은 다른 과정이다. 숫자를 범주로 바꾸는 이 과정은 수많은 제어 시스템에서 사용된다. 임계값을 잘 설정하는 것이 시스템의 반응성과 안정성을 결정한다.

## 미션 3. 픽셀 하나를 기울기 방향으로 움직여라

### 문제

LED 매트릭스 중앙(8, 8)에서 시작한 픽셀 하나가 기울기 방향으로 한 칸씩 이동하게 만들어라.

### 생각해 보기

픽셀의 좌표를 변수 `x, y`로 관리한다. 방향에 따라 좌표를 ±1 변경하되, 화면 밖으로 나가지 않도록 범위를 제한한다.

```
RIGHT → x += 1  (0 ≤ x ≤ 15 로 제한)
LEFT  → x -= 1
DOWN  → y += 1  (0 ≤ y ≤ 15 로 제한)
UP    → y -= 1
```

그리기 순서도 중요하다. 이동 전 위치를 끄고, 이동 후 위치를 켜야 한다.

### 코드

파일 이름: `m3_one_particle.py`

```python
# m3_one_particle.py
from mpu6050 import MPU6050
from ws2812 import WS2812Matrix
from machine import I2C, Pin
import utime

imu = MPU6050(I2C(0, scl=Pin(13), sda=Pin(12), freq=400000))
m   = WS2812Matrix([(0, 1)])

def get_direction(ax, ay):
    if abs(ax) > abs(ay):
        return "RIGHT" if ax > 0 else "LEFT"
    else:
        return "DOWN" if ay > 0 else "UP"

x, y = 8, 8   # 시작 위치

while True:
    ax, ay, _ = imu.accel
    direction = get_direction(ax, ay)

    # 이전 위치 끄기
    m[x, y].value = (0, 0, 0)

    # 이동
    if direction == "RIGHT":
        x = min(15, x + 1)
    elif direction == "LEFT":
        x = max(0, x - 1)
    elif direction == "DOWN":
        y = min(15, y + 1)
    else:  # UP
        y = max(0, y - 1)

    # 새 위치 켜기
    m[x, y].value = (0, 200, 100)
    m.update()

    utime.sleep_ms(80)
```

```sh
replx m3_one_particle.py
```

### 관찰하고 실험해 보자

- 보드를 기울이면 점이 기울기 방향으로 이동하는가?
- 화면 끝에 닿으면 멈추는가?
- `sleep_ms` 값을 줄이면 더 빠르게 움직이는가?
- 대각선 이동도 하게 만들려면 코드를 어떻게 바꿔야 하는가?

### 알게 된 것

픽셀을 이동시킬 때 "이전 위치를 끄고 새 위치를 켠다"는 순서가 중요하다. 이것이 잘못되면 잔상이 남는다. 게임에서 캐릭터가 이동하는 것도 같은 원리이다. 화면을 매 프레임마다 지우고 다시 그린다.

## 미션 4. 여러 입자가 아래로 떨어지고 쌓이게 하라

### 문제

5개의 입자를 LED 매트릭스의 랜덤 위치에 배치하고, 모두 아래로 떨어지도록 만들어라. 아래쪽 행이나 이미 다른 입자가 있으면 멈추어야 한다.

### 생각해 보기

여러 입자를 다루려면 배열이 필요하다. 각 입자의 좌표를 `(x, y)` 쌍으로 저장한다.

모래의 핵심 규칙은 다음과 같다.

```
아래 칸이 비어 있으면 → 아래로 한 칸 이동
아래 칸이 막혀 있으면 → 제자리에 머문다
```

**"막혀 있다"** 는 두 가지 경우이다.
1. 화면 맨 아래 행 (`y == 15`)
2. 이미 다른 입자가 그 칸에 있음

이것을 확인하려면 현재 입자들의 좌표 집합을 만들어 빠르게 검색할 수 있어야 한다.

### 코드

파일 이름: `m4_falling.py`

```python
# m4_falling.py
from ws2812 import WS2812Matrix
import urandom, utime

m = WS2812Matrix([(0, 1)])

# 5개 입자 초기화 (상단 랜덤 위치)
particles = [(urandom.randint(0, 15), urandom.randint(0, 3)) for _ in range(5)]

def occupied(x, y, particles):
    """(x, y) 위치에 입자가 있으면 True를 반환한다."""
    return (x, y) in particles

def update(particles):
    new_particles = []
    occupied_set  = set(particles)

    for (x, y) in particles:
        # 아래로 이동 가능한지 확인
        if y < 15 and (x, y + 1) not in occupied_set:
            new_particles.append((x, y + 1))
            occupied_set.discard((x, y))
            occupied_set.add((x, y + 1))
        else:
            new_particles.append((x, y))

    return new_particles

def draw(particles):
    m.clear()
    for (x, y) in particles:
        m[x, y].value = (200, 120, 20)   # 모래색
    m.update()

while True:
    particles = update(particles)
    draw(particles)
    utime.sleep_ms(100)
```

```sh
replx m4_falling.py
```

### 관찰하고 실험해 보자

- 입자들이 아래로 떨어지다 바닥에 쌓이는가?
- 두 입자가 같은 위치에 겹치지는 않는가?
- `urandom.randint(0, 15)` 범위를 좁히면 무슨 일이 생기는가?
- 입자 수를 20개로 늘리면 어떻게 되는가?

### 알게 된 것

`set(particles)`을 사용하면 특정 위치에 입자가 있는지 즉시 확인할 수 있다. 리스트에서 `in`으로 검색하는 것보다 집합(set)이 훨씬 빠르다. 이처럼 **올바른 자료 구조를 선택하는 것**이 성능에 큰 영향을 준다.

## 미션 5. 중력 방향이 바뀌게 하라

### 문제

IMU 기울기에 따라 중력 방향이 바뀌도록 미션 4의 코드를 수정하라. 보드를 기울이면 모래가 그 방향으로 흘러내려야 한다.

### 생각해 보기

미션 4에서는 중력 방향이 항상 아래(`y + 1`)로 고정되어 있었다. 이제 방향을 변수로 만든다.

```python
# 방향에 따른 이동량 (dx, dy)
GRAVITY = {
    "DOWN":  ( 0, +1),
    "UP":    ( 0, -1),
    "RIGHT": (+1,  0),
    "LEFT":  (-1,  0),
}
```

### 코드

파일 이름: `m5_gravity.py`

```python
# m5_gravity.py
from mpu6050 import MPU6050
from ws2812 import WS2812Matrix
from machine import I2C, Pin
import urandom, utime

imu = MPU6050(I2C(0, scl=Pin(13), sda=Pin(12), freq=400000))
m   = WS2812Matrix([(0, 1)])

GRAVITY = {
    "DOWN":  ( 0, +1),
    "UP":    ( 0, -1),
    "RIGHT": (+1,  0),
    "LEFT":  (-1,  0),
}

def get_direction(ax, ay):
    if abs(ax) > abs(ay):
        return "RIGHT" if ax > 0 else "LEFT"
    else:
        return "DOWN" if ay > 0 else "UP"

# 중앙 근처에 입자 20개 배치
particles = [(urandom.randint(4, 11), urandom.randint(4, 11)) for _ in range(20)]

def update(particles, direction):
    dx, dy     = GRAVITY[direction]
    new_parts  = []
    occupied   = set(particles)

    for (x, y) in particles:
        nx, ny = x + dx, y + dy
        # 화면 안에 있고, 해당 칸이 비어 있으면 이동
        if 0 <= nx <= 15 and 0 <= ny <= 15 and (nx, ny) not in occupied:
            new_parts.append((nx, ny))
            occupied.discard((x, y))
            occupied.add((nx, ny))
        else:
            new_parts.append((x, y))

    return new_parts

def draw(particles):
    m.clear()
    for (x, y) in particles:
        m[x, y].value = (200, 120, 20)
    m.update()

while True:
    ax, ay, _ = imu.accel
    direction = get_direction(ax, ay)
    particles = update(particles, direction)
    draw(particles)
    utime.sleep_ms(60)
```

## 최종 프로젝트. 모래 효과를 완성하라

### 추가할 개선 사항

1. **대각선 미끄러짐**: 아래가 막혀 있으면 대각선 아래로 이동을 시도한다. 실제 모래가 더 자연스럽게 쌓이는 원리이다.
2. **색상 변화**: 각 입자에 고유한 색상을 부여한다.
3. **입자 계속 추가**: 위쪽에서 주기적으로 새 입자가 생성된다.

### 최종 코드

파일 이름: `sand.py`

```python
# sand.py
from mpu6050 import MPU6050
from ws2812 import WS2812Matrix
from machine import I2C, Pin
import urandom, utime

imu = MPU6050(I2C(0, scl=Pin(13), sda=Pin(12), freq=400000))
m   = WS2812Matrix([(0, 1)])

def get_direction(ax, ay):
    if abs(ax) > abs(ay):
        return "RIGHT" if ax > 0 else "LEFT"
    else:
        return "DOWN" if ay > 0 else "UP"

# 입자: (x, y, r, g, b)
particles = []
MAX_P     = 80
FRAME     = 0

def add_particle():
    """현재 방향의 "위"에서 새 입자를 추가한다."""
    x = urandom.randint(0, 15)
    y = urandom.randint(0, 3)
    r = urandom.randint(150, 255)
    g = urandom.randint(80,  160)
    b = urandom.randint(10,  50)
    particles.append([x, y, r, g, b])

def update(direction):
    MOVES = {
        "DOWN":  [( 0,+1), (-1,+1), (+1,+1)],
        "UP":    [( 0,-1), (-1,-1), (+1,-1)],
        "RIGHT": [(+1, 0), (+1,-1), (+1,+1)],
        "LEFT":  [(-1, 0), (-1,-1), (-1,+1)],
    }
    occupied = {(p[0], p[1]) for p in particles}

    for p in particles:
        x, y = p[0], p[1]
        for dx, dy in MOVES[direction]:
            nx, ny = x + dx, y + dy
            if 0 <= nx <= 15 and 0 <= ny <= 15 and (nx, ny) not in occupied:
                occupied.discard((x, y))
                occupied.add((nx, ny))
                p[0], p[1] = nx, ny
                break

def draw():
    m.clear()
    for p in particles:
        m[p[0], p[1]].value = (p[2], p[3], p[4])
    m.update()

# 초기 입자
for _ in range(30):
    add_particle()

while True:
    ax, ay, _  = imu.accel
    direction  = get_direction(ax, ay)

    # 2프레임마다 새 입자 추가
    FRAME += 1
    if FRAME % 2 == 0 and len(particles) < MAX_P:
        add_particle()

    update(direction)
    draw()
    utime.sleep_ms(50)
```


## 돌아보기

모래 효과는 단 3가지 규칙으로 만들어진다.

1. 중력 방향으로 이동을 시도한다.
2. 그 칸이 막혀 있으면 대각선을 시도한다.
3. 모두 막혀 있으면 그 자리에 멈춘다.

이 단순한 규칙들이 합쳐져 복잡해 보이는 모래 흐름이 만들어진다. **복잡한 현상도 단순한 규칙의 반복에서 나온다**는 것이 이 장의 핵심 경험이다.
