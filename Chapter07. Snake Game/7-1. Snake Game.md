# 스네이크 게임

## 학습 목표

- 버튼 입력으로 이동 방향을 바꾸는 코드를 작성할 수 있다.
- 리스트로 뱀의 몸통을 관리하고 이동·성장·충돌을 구현할 수 있다.
- 게임 루프(입력 → 업데이트 → 출력)를 이해하고 직접 설계할 수 있다.

---

## 이 장의 도전 문제

> 방향 버튼으로 뱀을 조종하고, 먹이를 먹으면 길어지며, 벽이나 자기 몸에 닿으면 게임이 끝나는 스네이크 게임을 만들어 보자.

모든 게임은 같은 구조를 가진다. **입력 → 업데이트 → 화면 출력**을 매 프레임 반복하는 것이다. 이 장에서는 이 구조를 직접 경험하면서, 게임을 만드는 방법을 배운다.

## 미션 1. 버튼을 눌렀을 때 방향을 출력하라

### 문제

4개의 버튼 중 하나를 누르면 해당 방향 문자열을 터미널에 출력하라.

### 생각해 보기

`Pin.IN` 모드로 설정된 핀은 버튼이 눌리면 1, 떼면 0을 반환한다. 네 버튼 모두를 리스트에 담아두면 반복문으로 한 번에 처리할 수 있다.

### 코드

파일 이름: `m1_buttons.py`

```python
# m1_buttons.py
from machine import Pin
import utime

BUTTONS = {
    "UP":    Pin(16, Pin.IN),
    "LEFT":  Pin(17, Pin.IN),
    "DOWN":  Pin(18, Pin.IN),
    "RIGHT": Pin(19, Pin.IN),
}

prev = {name: 0 for name in BUTTONS}

while True:
    for name, pin in BUTTONS.items():
        cur = pin.value()
        if cur == 1 and prev[name] == 0:
            print("방향:", name)
        prev[name] = cur
    utime.sleep_ms(20)
```

```sh
replx m1_buttons.py
```

### 관찰하고 실험해 보자

- 각 버튼을 눌렀을 때 올바른 방향이 출력되는가?
- 버튼을 꾹 누르고 있으면 여러 번 출력되는가, 한 번만 출력되는가?
- 두 버튼을 동시에 누르면 어떻게 출력되는가?

### 알게 된 것

"새로 눌린 순간"을 감지하는 것이 핵심이다. `cur == 1 and prev == 0` 조건은 "이전에는 안 눌렸다가 지금 눌린 경우"만 처리한다. 이것을 **에지 검출(edge detection)** 이라고 한다.

## 미션 2. 픽셀 하나가 버튼 방향으로 움직이게 하라

### 문제

LED 매트릭스 중앙에 있는 픽셀이 버튼 방향으로 한 칸씩 움직이게 만들어라. 화면 끝에 닿으면 반대쪽으로 넘어간다(환형 이동).

### 생각해 보기

스네이크 게임의 화면은 환형(toroidal)이다. 왼쪽 끝에서 한 칸 더 왼쪽으로 가면 오른쪽 끝으로 나타난다. 파이썬에서는 `% 16` (나머지 연산)으로 이것을 구현할 수 있다.

```python
x = (x + dx) % 16   # 16이 되면 0으로, -1이면 15로
```

### 코드

파일 이름: `m2_move_pixel.py`

```python
# m2_move_pixel.py
from machine import Pin
from ws2812 import WS2812Matrix
import utime

BUTTONS = {
    "UP":    Pin(16, Pin.IN),
    "LEFT":  Pin(17, Pin.IN),
    "DOWN":  Pin(18, Pin.IN),
    "RIGHT": Pin(19, Pin.IN),
}
DIRS = {"UP": (0,-1), "LEFT": (-1,0), "DOWN": (0,1), "RIGHT": (1,0)}

m    = WS2812Matrix([(0, 1)])
x, y = 8, 8
prev = {name: 0 for name in BUTTONS}
direction = "RIGHT"

while True:
    # 버튼 입력
    for name, pin in BUTTONS.items():
        cur = pin.value()
        if cur == 1 and prev[name] == 0:
            direction = name
        prev[name] = cur

    # 이전 위치 끄기
    m[x, y].value = (0, 0, 0)

    # 이동
    dx, dy = DIRS[direction]
    x = (x + dx) % 16
    y = (y + dy) % 16

    # 새 위치 켜기
    m[x, y].value = (0, 200, 0)
    m.update()

    utime.sleep_ms(150)
```

## 미션 3. 뱀에 꼬리를 붙여라

### 문제

뱀이 이동할 때 몸통을 길이 5로 유지하면서 움직이게 만들어라.

### 생각해 보기

뱀의 몸통은 좌표의 **리스트**이다. 머리가 새로운 위치로 이동하면 리스트 앞에 추가하고, 꼬리(마지막 원소)를 제거하면 길이가 유지된다.

```
이동 전: [(3,5), (2,5), (1,5)]   ← 머리가 (3,5)
이동 후: [(4,5), (3,5), (2,5)]   ← (1,5) 꼬리 제거
```

파이썬에서는 `deque` 대신 리스트의 `insert(0, new_head)`와 `pop()` 으로 구현한다.

### 코드

파일 이름: `m3_snake_body.py`

```python
# m3_snake_body.py
from machine import Pin
from ws2812 import WS2812Matrix
import utime

BUTTONS = {
    "UP":    Pin(16, Pin.IN),
    "LEFT":  Pin(17, Pin.IN),
    "DOWN":  Pin(18, Pin.IN),
    "RIGHT": Pin(19, Pin.IN),
}
DIRS = {"UP": (0,-1), "LEFT": (-1,0), "DOWN": (0,1), "RIGHT": (1,0)}

m         = WS2812Matrix([(0, 1)])
snake     = [(8, 8), (7, 8), (6, 8), (5, 8), (4, 8)]   # 초기 몸통 5칸
direction = "RIGHT"
prev      = {name: 0 for name in BUTTONS}

def draw(snake):
    m.clear()
    for i, (x, y) in enumerate(snake):
        if i == 0:
            m[x, y].value = (0, 255, 0)    # 머리: 밝은 초록
        else:
            m[x, y].value = (0, 100, 0)    # 몸통: 어두운 초록
    m.update()

while True:
    for name, pin in BUTTONS.items():
        cur = pin.value()
        if cur == 1 and prev[name] == 0:
            direction = name
        prev[name] = cur

    # 새 머리 위치 계산
    hx, hy = snake[0]
    dx, dy = DIRS[direction]
    new_head = ((hx + dx) % 16, (hy + dy) % 16)

    # 몸통 업데이트 (앞에 추가, 뒤에서 제거)
    snake.insert(0, new_head)
    snake.pop()

    draw(snake)
    utime.sleep_ms(150)
```

```sh
replx m3_snake_body.py
```

### 관찰하고 실험해 보자

- 뱀이 5칸 길이를 유지하면서 움직이는가?
- 방향 전환이 부드럽게 되는가?
- 역방향 버튼을 누르면 어떻게 되는가? (올바른 스네이크 게임에서는 역방향이 금지된다. 어떻게 구현해야 하는가?)

### 알게 된 것

뱀의 이동은 "리스트 맨 앞에 추가하고, 맨 뒤를 제거하는" 연산의 조합이다. 먹이를 먹어 길어지는 것은 `pop()`을 한 번 생략하는 것으로 쉽게 구현할 수 있다.

## 미션 4. 먹이를 만들고 먹으면 길어지게 하라

### 문제

랜덤 위치에 먹이(빨간 픽셀)를 배치하고, 뱀이 먹이 위치에 도달하면 길이가 1 증가하고 새 먹이가 생성되게 만들어라.

### 생각해 보기

먹이를 먹었는지 확인하는 방법은 간단하다. **새 머리 위치가 먹이 위치와 같으면** 먹은 것이다.

먹으면 길어지는 것은 `pop()`을 실행하지 않으면 된다. 꼬리를 제거하지 않으면 몸통이 그대로 남아 길이가 1 증가한다.

새 먹이는 뱀의 몸통과 겹치지 않는 위치에 배치해야 한다.

### 코드

파일 이름: `m4_food.py`

```python
# m4_food.py
from machine import Pin
from ws2812 import WS2812Matrix
import urandom, utime

BUTTONS = {
    "UP":    Pin(16, Pin.IN),
    "LEFT":  Pin(17, Pin.IN),
    "DOWN":  Pin(18, Pin.IN),
    "RIGHT": Pin(19, Pin.IN),
}
DIRS = {"UP": (0,-1), "LEFT": (-1,0), "DOWN": (0,1), "RIGHT": (1,0)}

m         = WS2812Matrix([(0, 1)])
snake     = [(8, 8), (7, 8), (6, 8)]
direction = "RIGHT"
prev      = {name: 0 for name in BUTTONS}

def random_food(snake):
    """뱀 몸통과 겹치지 않는 랜덤 위치를 반환한다."""
    snake_set = set(snake)
    while True:
        pos = (urandom.randint(0, 15), urandom.randint(0, 15))
        if pos not in snake_set:
            return pos

food = random_food(snake)

def draw(snake, food):
    m.clear()
    fx, fy = food
    m[fx, fy].value = (200, 0, 0)   # 먹이: 빨강
    for i, (x, y) in enumerate(snake):
        m[x, y].value = (0, 255, 0) if i == 0 else (0, 120, 0)
    m.update()

while True:
    for name, pin in BUTTONS.items():
        cur = pin.value()
        if cur == 1 and prev[name] == 0:
            direction = name
        prev[name] = cur

    hx, hy   = snake[0]
    dx, dy   = DIRS[direction]
    new_head = ((hx + dx) % 16, (hy + dy) % 16)

    snake.insert(0, new_head)

    if new_head == food:
        # 먹이를 먹었다: 꼬리 제거 안 함 → 길이 증가
        food = random_food(snake)
        print("먹이 먹음! 현재 길이:", len(snake))
    else:
        snake.pop()   # 정상 이동

    draw(snake, food)
    utime.sleep_ms(150)
```


## 최종 프로젝트. 충돌 감지와 게임 오버를 추가하라

### 추가할 기능

1. **충돌 감지**: 새 머리가 자기 몸통과 겹치면 게임 오버이다.
2. **게임 오버 표시**: 화면이 전부 빨간색으로 바뀐 뒤 게임이 재시작된다.
3. **역방향 금지**: 180도 반대 방향 입력은 무시한다.
4. **속도 점진 증가**: 길이가 늘어날수록 이동 속도가 빨라진다.

### 최종 코드

파일 이름: `snake.py`

```python
# snake.py
from machine import Pin
from ws2812 import WS2812Matrix
import urandom, utime

BUTTONS = {
    "UP":    Pin(16, Pin.IN),
    "LEFT":  Pin(17, Pin.IN),
    "DOWN":  Pin(18, Pin.IN),
    "RIGHT": Pin(19, Pin.IN),
}
DIRS = {"UP": (0,-1), "LEFT": (-1,0), "DOWN": (0,1), "RIGHT": (1,0)}
OPPOSITE = {"UP": "DOWN", "DOWN": "UP", "LEFT": "RIGHT", "RIGHT": "LEFT"}

m = WS2812Matrix([(0, 1)])

def random_food(snake):
    s = set(snake)
    while True:
        pos = (urandom.randint(0, 15), urandom.randint(0, 15))
        if pos not in s:
            return pos

def draw(snake, food):
    m.clear()
    fx, fy = food
    m[fx, fy].value = (180, 0, 0)
    for i, (x, y) in enumerate(snake):
        m[x, y].value = (0, 220, 0) if i == 0 else (0, 90, 0)
    m.update()

def game_over_anim():
    for _ in range(3):
        for x in range(16):
            for y in range(16):
                m[x, y].value = (150, 0, 0)
        m.update()
        utime.sleep_ms(200)
        m.clear()
        m.update()
        utime.sleep_ms(200)

def init():
    return [(8, 8), (7, 8), (6, 8)], "RIGHT", random_food([(8,8),(7,8),(6,8)])

snake, direction, food = init()
prev  = {name: 0 for name in BUTTONS}
score = 0

while True:
    # 1. 입력
    for name, pin in BUTTONS.items():
        cur = pin.value()
        if cur == 1 and prev[name] == 0:
            if name != OPPOSITE[direction]:   # 역방향 금지
                direction = name
        prev[name] = cur

    # 2. 업데이트
    hx, hy   = snake[0]
    dx, dy   = DIRS[direction]
    new_head = ((hx + dx) % 16, (hy + dy) % 16)

    # 자기 몸통 충돌
    if new_head in set(snake):
        print(f"게임 오버! 점수: {score}")
        game_over_anim()
        snake, direction, food = init()
        score = 0
        utime.sleep_ms(500)
        continue

    snake.insert(0, new_head)

    if new_head == food:
        score += 1
        food   = random_food(snake)
        print("점수:", score)
    else:
        snake.pop()

    # 3. 출력
    draw(snake, food)

    # 길이에 따라 속도 증가
    delay = max(60, 200 - score * 5)
    utime.sleep_ms(delay)
```

## 돌아보기

스네이크 게임을 만들면서 사용한 기법을 정리해 보자.

| 미션 | 핵심 기법 |
|------|-----------|
| 미션 1 | 에지 검출 (눌린 순간만 감지) |
| 미션 2 | 나머지 연산으로 환형 이동 구현 |
| 미션 3 | 리스트 insert/pop으로 뱀 몸통 관리 |
| 미션 4 | 충돌 감지 = 위치 동등 비교 |
| 최종 | 입력 → 업데이트 → 출력 게임 루프 |

이 구조는 모든 게임의 기본이다. 테트리스, 팩맨, 모두 같은 루프로 동작한다.
