# 나만의 BLE 스피커

## 1. 실습 주제

커스텀 UUID 기반 명령으로 **Bluetooth 스피커(앰프) 원격 제어** 만들기

## 2. 프로젝트 개요

Pico 보드가 자체적으로 광고(Advertise)하고, 연결된 중앙장치(스마트폰 등)로부터 **문자 명령**을 받아 앰프의 기능(볼륨 업/다운, 다음/이전 트랙)을 실행합니다.
명령을 처리한 뒤에는 응답(ACK/NACK)을 특성 값으로 갱신하고 알림(Notify)로 보내, 중앙장치가 결과를 즉시 확인할 수 있게 합니다.

## 3. 개념 설명 (BLE 개념 제외)

* **명령 처리 흐름**: 중앙장치가 텍스트(예: `"up"`, `"down"`)를 쓰면, 보드는 이를 소문자/공백 제거 후 해석합니다.
  매칭되는 기능을 실행하고, 실행 결과를 `"ACK:<cmd>"` 형태로 작성해 동일 특성에 기록합니다.
* **결과 전달**: 최근 연결된 모든 중앙장치에 **알림(Notify)** 를 보내, 응답 문자열을 즉시 전달합니다.
* **연결/해제 상태 관리**: 연결 이벤트에 따라 연결 집합을 업데이트하고, 해제 시에는 자동으로 재광고를 시작합니다.

## 4. Pico 코드

```python
import ubluetooth as bt
import uasyncio as asyncio
import struct
from machine import Pin, I2S

_IRQ_CENTRAL_CONNECT=1
_IRQ_CENTRAL_DISCONNECT=2
_IRQ_GATTS_WRITE=3

SVC_UUID=bt.UUID('6E400001-B5A3-F393-E0A9-E50E24DCCA9E')
CHAR_UUID=bt.UUID('6E400002-B5A3-F393-E0A9-E50E24DCCA9E')
FLAGS=bt.FLAG_READ|bt.FLAG_WRITE|bt.FLAG_NOTIFY

SCK_PIN=4
WS_PIN=5
SD_PIN=7

ble=bt.BLE()
ble.active(True)

((CHAR_H,),)=ble.gatts_register_services(((SVC_UUID,((CHAR_UUID,FLAGS),)),))

conns=set()
cmd_pending=0

play_evt=asyncio.Event()
play_evt.set()

def adv_start():
    adv=bytearray()
    adv+=struct.pack('BB',2,0x01)+b'\x06'
    name=b'Pico-CTL'
    adv+=struct.pack('BB',len(name)+1,0x09)+name
    ble.gap_advertise(100_000, adv_data=adv)

def irq(e,d):
    global cmd_pending
    if e==_IRQ_CENTRAL_CONNECT:
        conns.add(d[0])
    elif e==_IRQ_CENTRAL_DISCONNECT:
        conns.discard(d[0])
        adv_start()
    elif e==_IRQ_GATTS_WRITE and d[1]==CHAR_H:
        cmd_pending=1

def wav_open(path):
    f=open(path,"rb")
    if f.read(4)!=b"RIFF":
        f.close()
        raise ValueError("riff")
    f.read(4)
    if f.read(4)!=b"WAVE":
        f.close()
        raise ValueError("wave")
    fmt=None
    data=None
    while True:
        h=f.read(8)
        if len(h)<8:
            break
        cid,csz=struct.unpack("<4sI",h)
        if cid==b"fmt ":
            b=f.read(csz)
            if len(b)<16:
                f.close()
                raise ValueError("fmt")
            af,nch,rate,br,ba,bits=struct.unpack_from("<HHIIHH",b,0)
            fmt=(af,nch,rate,bits)
        elif cid==b"data":
            data_start=f.tell()
            data_len=csz
            data=(data_start,data_len)
            f.seek(csz,1)
        else:
            f.seek(csz,1)
        if csz & 1:
            f.seek(1,1)
        if fmt and data:
            break
    if not fmt or not data:
        f.close()
        raise ValueError("chunks")
    af,nch,rate,bits=fmt
    if af!=1 or bits!=16 or (nch!=1 and nch!=2):
        f.close()
        raise ValueError("pcm16")
    return f,rate,nch,data[0],data[1]

async def cmd_loop():
    global cmd_pending
    while True:
        if cmd_pending:
            cmd_pending=0
            v=ble.gatts_read(CHAR_H)
            try:
                s=v.decode().strip().lower()
            except:
                s=""
            if s in ("play","resume","start"):
                play_evt.set()
                resp=b"ACK:play"
            elif s in ("pause","stop"):
                play_evt.clear()
                resp=b"ACK:pause"
            elif s=="toggle":
                if play_evt.is_set():
                    play_evt.clear()
                    resp=b"ACK:pause"
                else:
                    play_evt.set()
                    resp=b"ACK:play"
            else:
                resp=b"NACK"
            ble.gatts_write(CHAR_H, resp)
            for c in tuple(conns):
                try:
                    ble.gatts_notify(c, CHAR_H)
                except:
                    pass
        await asyncio.sleep_ms(20)

async def audio_loop():
    f,rate,nch,data_start,data_len=wav_open("test.wav")
    fmt=I2S.MONO if nch==1 else I2S.STEREO
    audio=I2S(
        0,
        sck=Pin(SCK_PIN),
        ws=Pin(WS_PIN),
        sd=Pin(SD_PIN),
        mode=I2S.TX,
        bits=16,
        format=fmt,
        rate=rate,
        ibuf=8192,
    )
    buf=bytearray(4096)
    mv=memoryview(buf)
    while True:
        await play_evt.wait()
        f.seek(data_start)
        remaining=data_len
        while remaining>0:
            if not play_evt.is_set():
                break
            n=4096 if remaining>=4096 else remaining
            b=f.readinto(mv[:n])
            if not b:
                break
            remaining-=b
            audio.write(mv[:b])
            await asyncio.sleep_ms(0)

ble.irq(irq)
adv_start()

async def main():
    asyncio.create_task(cmd_loop())
    asyncio.create_task(audio_loop())
    while True:
        await asyncio.sleep_ms(1000)

asyncio.run(main())

```

## 5. 코드 요약 (Pico)

### 5-1) 서비스·특성 등록과 광고 시작

```python
SVC_UUID=bt.UUID('...0001...')
CHAR_UUID=bt.UUID('...0002...')
FLAGS=bt.FLAG_READ|bt.FLAG_WRITE|bt.FLAG_NOTIFY
((CHAR_H,),)=ble.gatts_register_services(((SVC_UUID,((CHAR_UUID,FLAGS),)),))
```

* 한 개의 서비스와 한 개의 특성을 등록합니다.
  특성은 **읽기/쓰기/알림**을 모두 허용합니다.
* `adv_start()`에서 장치 이름(`Pico-CTL`)을 포함한 광고 패킷을 구성해 주기적으로 송출합니다.

### 5-2) 연결 상태 관리

```python
conns=set()
def irq(e,d):
    if e==_IRQ_CENTRAL_CONNECT:   conns.add(d[0])
    elif e==_IRQ_CENTRAL_DISCONNECT:
        conns.discard(d[0]); adv_start()
```

* 연결되면 연결 핸들을 집합에 저장합니다.
* 해제되면 집합에서 제거하고 **즉시 재광고**를 시작합니다(다음 연결 대기).

### 5-3) 명령 수신 → 앰프 제어 → 응답 전송

```python
elif e==_IRQ_GATTS_WRITE and d[1]==CHAR_H and conns:
    v=ble.gatts_read(CHAR_H)
    s=v.decode().strip().lower()  # 예: "up", "down"
    resp=handle_cmd(s)            # 실행 후 응답 바이트 반환
    ble.gatts_write(CHAR_H, resp) # 응답을 특성 값에 기록
    for c in tuple(conns):
        ble.gatts_notify(c, CHAR_H)  # 모든 연결 대상에게 알림
```

* 중앙장치가 특성에 문자열을 **쓰기** 하면, 그 값을 읽어 공백 제거·소문자로 정규화합니다.
* `handle_cmd()`가 해당 문자열을 해석해 **앰프 제어 함수**를 호출하고,
  성공 시 `b"ACK:<cmd>"`, 미지원 명령 시 `b"NACK"`을 반환합니다.
* 응답을 특성에 기록하고, **알림(Notify)** 로 즉시 전송합니다.

### 5-4) 앰프 제어 명령 매핑

```python
def handle_cmd(cmd):
    if cmd=='up': amp.up();   return b'ACK:up'
    if cmd=='down': amp.down(); return b'ACK:down'
    if cmd=='next': amp.next(); return b'ACK:next'
    if cmd=='prev': amp.prev(); return b'ACK:prev'
    return b'NACK'
```

* 지원 명령: `up`, `down`, `next`, `prev`
* 각 명령은 `BtAudioAmp` 의 대응 메서드를 호출합니다.

## 6. 결과 확인

* 스마트폰/PC의 BLE GATT 도구에서 장치 **Pico-CTL**에 연결합니다.
* 특성에 `"up"`, `"down"`, `"next"`, `"prev"` 중 하나를 **Write** 하면, 스피커가 동작하고 특성 값이 `ACK:*` 으로 갱신됩니다.
* Notify 수신이 설정되어 있으면 응답이 즉시 표시됩니다.
