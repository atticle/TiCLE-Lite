# 소리 스펙트럼 분석기

## 학습 목표

- 마이크에서 들어온 소리 데이터를 숫자로 읽고 출력할 수 있다.
- RMS로 소리의 전체 크기를 계산해 LED 막대로 표현할 수 있다.
- Goertzel 알고리즘으로 특정 주파수 성분의 크기를 구할 수 있다.
- 16개 주파수 대역을 LED 열로 나타내는 스펙트럼 분석기를 완성할 수 있다.

---

## 이 장의 도전 문제

> 소리는 들리는데, 어떤 높이의 소리가 강한지 눈으로는 볼 수 없다.
> 마이크로 들어온 소리를 LED 화면에 스펙트럼으로 표시해 보자.

음악 플레이어의 이퀄라이저 화면처럼, 낮은 소리는 왼쪽 열에, 높은 소리는 오른쪽 열에 막대로 나타나는 장치이다.

---

## 사전 지식. 소리는 숫자다

마이크는 공기의 진동을 전기 신호로 바꾼다. TiCLE Lite의 디지털 마이크(MEMS 마이크)는 그 전기 신호를 다시 숫자로 바꾸어 준다. 이 숫자들의 연속이 바로 소리 데이터이다.

초당 8000개의 숫자를 읽는다고 하면, 1초의 소리가 8000개의 정수 배열이 된다. 이것이 **샘플링**이다.

```
마이크 → [숫자, 숫자, 숫자, ...] → 분석 → LED 시각화
        (초당 8000개)
```

이 장에서는 그 숫자들로 두 가지를 계산한다.

- **전체 크기(RMS)**: 지금 소리가 얼마나 큰가?
- **주파수별 크기(Goertzel)**: 지금 소리 안에서 어떤 높이가 얼마나 강한가?

---

## 미션 1. 마이크에서 숫자를 읽어라

### 문제

마이크로 소리를 100ms 단위로 읽고, 첫 번째 샘플 값을 터미널에 출력하라.

### 생각해 보기

TiCLE Lite의 마이크는 I2S 프로토콜로 연결되어 있다. I2S는 오디오 장치가 숫자 데이터를 주고받을 때 쓰는 방식이다.

마이크 핀 연결:

| 신호 | 핀 번호 | 역할 |
|------|---------|------|
| SCK | 4 | 비트 클럭 |
| WS | 5 | 채널 선택 (좌/우) |
| SD | 6 | 데이터 출력 |

각 샘플은 32비트(4바이트)이지만 실제 오디오 데이터는 상위 18비트에만 담긴다. 읽은 값을 14만큼 오른쪽으로 비트 이동(>>)하면 일반적인 정수 범위 값이 된다.

### 코드

파일 이름: `m1_mic_read.py`

```python
# m1_mic_read.py
from machine import Pin, I2S
import struct

# I2S 마이크 초기화
mic = I2S(
    0,
    sck=Pin(4), ws=Pin(5), sd=Pin(6),
    mode=I2S.RX,
    bits=32,
    format=I2S.MONO,
    rate=8000,
    ibuf=4096
)

# 한 번에 읽을 샘플 수
N_SAMPLES = 128
buf = bytearray(N_SAMPLES * 4)   # 샘플 1개 = 4바이트

while True:
    mic.readinto(buf)             # 마이크에서 읽기

    # 32비트 정수로 변환 (little-endian, signed)
    samples = struct.unpack('<' + 'i' * N_SAMPLES, buf)

    # 상위 비트에 실제 데이터가 있으므로 14비트 오른쪽 이동
    values = [s >> 14 for s in samples]

    print("첫 샘플:", values[0], "  최솟값:", min(values), "  최댓값:", max(values))
```

```sh
replx m1_mic_read.py
```

### 관찰하고 실험해 보자

- 조용할 때 출력되는 값의 범위는 어느 정도인가?
- 마이크에 크게 말하거나 손뼉을 치면 최댓값/최솟값이 어떻게 변하는가?
- `N_SAMPLES = 512`로 바꾸면 읽기 속도가 어떻게 달라지는가? (더 빨라지는가, 느려지는가?)

### 알게 된 것

소리 데이터는 0을 기준으로 양수와 음수를 오가는 숫자의 흐름이다. 조용할 때는 0 근처에서 흔들리고, 소리가 클수록 진폭이 커진다. 이것이 파형(waveform)이다.

## 미션 2. 소리 크기를 하나의 숫자로 나타내라

### 문제

샘플 배열에서 소리의 전체 크기를 계산해 0~16 사이의 숫자로 출력하라.

### 생각해 보기

소리의 숫자들은 양수와 음수를 계속 오간다. 그냥 더하면 서로 상쇄되어 0이 된다. 크기를 구하려면 **제곱해서 평균을 낸 뒤 제곱근**을 취해야 한다. 이것이 RMS(Root Mean Square)이다.

$$\text{RMS} = \sqrt{\frac{1}{N}\sum_{i=1}^{N} x_i^2}$$

조용한 방에서도 마이크는 미세한 잡음을 잡는다. 그래서 0은 아니더라도 낮은 RMS 값이 항상 존재한다. 이것을 **바닥 잡음(noise floor)** 이라고 한다.

계산된 RMS 값이 특정 범위 안에 있다고 가정하고, 그것을 0~16 사이의 정수로 바꾸는 것이 이번 미션이다.

### 코드

파일 이름: `m2_rms.py`

```python
# m2_rms.py
from machine import Pin, I2S
import struct, math

mic = I2S(
    0,
    sck=Pin(4), ws=Pin(5), sd=Pin(6),
    mode=I2S.RX, bits=32, format=I2S.MONO,
    rate=8000, ibuf=4096
)

N       = 128
buf     = bytearray(N * 4)
MAX_LED = 16

def calc_rms(samples):
    mean = sum(samples) / len(samples)
    sq_sum = sum((s - mean) ** 2 for s in samples)
    return math.sqrt(sq_sum / len(samples))

while True:
    mic.readinto(buf)
    raw     = struct.unpack('<' + 'i' * N, buf)
    values  = [s >> 14 for s in raw]
    rms     = calc_rms(values)

    # RMS 값을 0~MAX_LED 범위로 변환
    # 조용할 때 약 10~50, 큰 소리일 때 약 2000~8000 (환경마다 다름)
    height = int(min(rms / 500.0, 1.0) * MAX_LED)

    print(f"RMS: {rms:6.1f}  높이: {height:2d}  {'|' * height}")
```

```sh
replx m2_rms.py
```

### 관찰하고 실험해 보자

- 조용한 환경에서 `height`는 보통 얼마인가?
- 손뼉을 치면 `height`가 얼마까지 올라가는가?
- 높이가 16을 넘어간다면 `500.0` 값을 얼마로 바꾸어야 하는가?
- `mean`을 빼고 계산(`mean = 0`)하면 결과가 달라지는가? 왜 그런가?

### 알게 된 것

RMS를 계산할 때 평균을 먼저 빼는 이유가 있다. 마이크 신호에는 DC 오프셋(0이 아닌 상수 성분)이 포함될 수 있다. 이것을 제거하지 않으면 소리가 없어도 RMS가 높게 나온다. 평균을 빼는 것은 신호에서 이 상수 성분을 제거하는 작업이다.

## 미션 3. 소리 크기를 LED 막대로 그려라

### 문제

RMS 계산 결과를 LED 매트릭스의 첫 번째 열(x=0)에 세로 막대로 표시하라.

### 생각해 보기

LED 매트릭스의 y 좌표는 위에서 아래로 증가한다. 막대를 아래에서 위로 채우려면 `y = 15 - i` 공식을 쓴다.

```
height = 3 일 때 표시되어야 할 픽셀:
y=13 → 초록 (15-2)
y=14 → 초록 (15-1)
y=15 → 초록 (15-0)
```

먼저 열 전체를 꺼(검정)두고, 그 다음 높이만큼만 켜야 한다. 그렇지 않으면 이전 프레임의 잔상이 남는다.

### 코드

파일 이름: `m3_vumeter.py`

```python
# m3_vumeter.py
from machine import Pin, I2S
from ws2812 import WS2812Matrix
import struct, math

mic = I2S(
    0,
    sck=Pin(4), ws=Pin(5), sd=Pin(6),
    mode=I2S.RX, bits=32, format=I2S.MONO,
    rate=8000, ibuf=4096
)

m   = WS2812Matrix([(0, 1)])   # pin=0, state machine=1
N   = 128
buf = bytearray(N * 4)

def calc_rms(samples):
    mean = sum(samples) / len(samples)
    sq   = sum((s - mean) ** 2 for s in samples)
    return math.sqrt(sq / len(samples))

def draw_col(x, height, color=(0, 200, 0)):
    """x열을 아래에서부터 height만큼 color로 채운다."""
    for y in range(16):
        m[x, y].value = (0, 0, 0)
    for i in range(height):
        m[x, 15 - i].value = color
    m.update()

while True:
    mic.readinto(buf)
    raw    = struct.unpack('<' + 'i' * N, buf)
    vals   = [s >> 14 for s in raw]
    rms    = calc_rms(vals)
    height = int(min(rms / 500.0, 1.0) * 16)
    draw_col(0, height)
```

```sh
replx m3_vumeter.py
```

### 관찰하고 실험해 보자

- 소리에 맞춰 막대가 자연스럽게 움직이는가?
- 소리가 없을 때 막대가 완전히 내려가는가?
- `color=(200, 0, 0)`으로 바꾸면 빨간 막대가 그려지는가?
- 막대를 위에서 아래로 채우려면 `15 - i` 부분을 어떻게 바꿔야 하는가?

### 알게 된 것

`draw_col()` 함수를 만든 이유가 있다. 다음 미션에서는 열이 여러 개가 된다. 같은 코드를 반복해서 쓰는 대신, 함수 하나에 정리해 두면 `draw_col(x, height)`처럼 간단하게 호출할 수 있다. 이처럼 **반복될 코드를 함수로 묶는 것**이 프로그래밍의 핵심 기법 중 하나이다.

## 미션 4. 낮은 소리와 높은 소리를 따로 구분하라

### 문제

같은 소리 데이터에서 낮은 소리(120 Hz)와 높은 소리(700 Hz)의 크기를 따로 계산해, 두 개의 열로 표시하라.

### 생각해 보기

"소리가 크다"와 "소리가 높다"는 다른 개념이다. 베이스 드럼과 심벌즈는 비슷한 크기일 수 있지만, 포함된 주파수 성분은 완전히 다르다.

특정 주파수 성분이 얼마나 포함됐는지 계산하는 방법이 **Goertzel 알고리즘**이다. 필요한 주파수 하나만 골라 계산하므로 전체 FFT보다 훨씬 가볍다.

원리는 간단하다. 분석하려는 주파수에 맞는 기준 파형을 만든 뒤, 입력 신호와 얼마나 비슷한지 비교한다. 비교가 잘 맞으면 그 주파수가 강하게 포함된 것이다.

```python
def goertzel(samples, target_freq, sample_rate):
    """samples 안에서 target_freq 성분의 세기를 반환한다."""
    N     = len(samples)
    k     = int(0.5 + N * target_freq / sample_rate)
    omega = 2.0 * math.pi * k / N
    coeff = 2.0 * math.cos(omega)
    s0, s1, s2 = 0.0, 0.0, 0.0
    for x in samples:
        s0 = x + coeff * s1 - s2
        s2 = s1
        s1 = s0
    power = s2 * s2 + s1 * s1 - coeff * s1 * s2
    return max(0.0, power)
```

수식을 외울 필요는 없다. **"이 함수를 쓰면 특정 주파수의 세기가 숫자로 나온다"** 는 것만 알면 된다.

### 코드

파일 이름: `m4_two_cols.py`

```python
# m4_two_cols.py
from machine import Pin, I2S
from ws2812 import WS2812Matrix
import struct, math

mic = I2S(
    0,
    sck=Pin(4), ws=Pin(5), sd=Pin(6),
    mode=I2S.RX, bits=32, format=I2S.MONO,
    rate=8000, ibuf=4096
)

m       = WS2812Matrix([(0, 1)])
N       = 256
RATE    = 8000
buf     = bytearray(N * 4)

def goertzel(samples, target_freq, sample_rate):
    n     = len(samples)
    k     = int(0.5 + n * target_freq / sample_rate)
    omega = 2.0 * math.pi * k / n
    coeff = 2.0 * math.cos(omega)
    s1 = s2 = 0.0
    for x in samples:
        s0 = x + coeff * s1 - s2
        s2 = s1
        s1 = s0
    return max(0.0, s2 * s2 + s1 * s1 - coeff * s1 * s2)

def draw_col(x, height, color):
    for y in range(16):
        m[x, y].value = (0, 0, 0)
    for i in range(height):
        m[x, 15 - i].value = color
    m.update()

while True:
    mic.readinto(buf)
    raw   = struct.unpack('<' + 'i' * N, buf)
    vals  = [s >> 14 for s in raw]

    low   = goertzel(vals, 120.0, RATE)   # 낮은 소리 성분
    high  = goertzel(vals, 700.0, RATE)   # 높은 소리 성분

    # 두 값 중 큰 것을 기준으로 정규화
    mx = max(low, high, 1.0)
    h_low  = int((low  / mx) * 16)
    h_high = int((high / mx) * 16)

    draw_col(0, h_low,  (0, 0, 200))    # 낮은 소리 → 파란색
    draw_col(1, h_high, (200, 0, 0))    # 높은 소리 → 빨간색
    m.update()
```

```sh
replx m4_two_cols.py
```

### 관찰하고 실험해 보자

- 낮은 목소리("아~")를 내면 파란색 막대(x=0)가 더 높이 올라가는가?
- 높은 음("쉬~")을 내면 빨간색 막대(x=1)가 더 높이 올라가는가?
- 두 가지 소리를 동시에 내면 두 막대가 함께 올라가는가?
- `120.0`과 `700.0`을 다른 주파수로 바꾸면 반응하는 소리가 달라지는가?

### 알게 된 것

Goertzel 함수는 주파수 하나당 하나의 숫자를 돌려준다. 이 숫자가 크면 그 주파수의 소리가 강하게 포함된 것이다. 두 값을 직접 비교하기 위해 최댓값으로 나누어 정규화했다. 이로써 두 막대의 상대적 크기를 비교할 수 있다.

## 미션 5. 여덟 개의 주파수 대역으로 나누다

### 문제

80 Hz ~ 900 Hz 범위를 로그 스케일로 8등분하고, 각 주파수의 세기를 8개 열로 표시하라.

### 생각해 보기

사람의 귀는 주파수를 로그 스케일로 인식한다. 100 Hz와 200 Hz의 차이를 1000 Hz와 1100 Hz의 차이와 같은 크기로 느낀다. 그래서 스펙트럼 분석기의 주파수는 일정 간격이 아닌 로그 간격으로 배치한다.

로그 간격으로 N개의 주파수를 만드는 방법:

```python
import math

def log_freqs(f_min, f_max, n):
    """f_min ~ f_max 사이를 로그 스케일로 n개 나눈다."""
    log_min = math.log(f_min)
    log_max = math.log(f_max)
    step    = (log_max - log_min) / (n - 1)
    return [math.exp(log_min + i * step) for i in range(n)]
```

### 코드

파일 이름: `m5_eight_cols.py`

```python
# m5_eight_cols.py
from machine import Pin, I2S
from ws2812 import WS2812Matrix
import struct, math

mic = I2S(
    0,
    sck=Pin(4), ws=Pin(5), sd=Pin(6),
    mode=I2S.RX, bits=32, format=I2S.MONO,
    rate=8000, ibuf=4096
)

m    = WS2812Matrix([(0, 1)])
N    = 256
RATE = 8000
COLS = 8
buf  = bytearray(N * 4)

def goertzel(samples, freq, rate):
    n     = len(samples)
    k     = int(0.5 + n * freq / rate)
    omega = 2.0 * math.pi * k / n
    coeff = 2.0 * math.cos(omega)
    s1 = s2 = 0.0
    for x in samples:
        s0 = x + coeff * s1 - s2
        s2 = s1
        s1 = s0
    return max(0.0, s2 * s2 + s1 * s1 - coeff * s1 * s2)

def log_freqs(f_min, f_max, n):
    lmin = math.log(f_min)
    lmax = math.log(f_max)
    step = (lmax - lmin) / (n - 1)
    return [math.exp(lmin + i * step) for i in range(n)]

FREQS = log_freqs(80.0, 900.0, COLS)
print("분석 주파수:", [int(f) for f in FREQS])

def draw_col(x, height):
    # 낮은 주파수 = 파랑, 높은 주파수 = 빨강 (그라데이션)
    r = int(255 * x / (COLS - 1))
    b = 255 - r
    color = (r, 0, b)
    for y in range(16):
        m[x, y].value = (0, 0, 0)
    for i in range(height):
        m[x, 15 - i].value = color
    m.update()

while True:
    mic.readinto(buf)
    raw  = struct.unpack('<' + 'i' * N, buf)
    vals = [s >> 14 for s in raw]

    powers = [goertzel(vals, f, RATE) for f in FREQS]
    mx     = max(powers + [1.0])

    for x, p in enumerate(powers):
        h = int((p / mx) * 16)
        draw_col(x, h)
    m.update()
```


- 낮은 음을 내면 왼쪽 열(파란색)이 움직이는가?
- 높은 음을 내면 오른쪽 열(빨간색)이 움직이는가?
- 음악을 틀었을 때 여러 열이 동시에 반응하는가?
- `COLS = 4`로 줄이면 속도가 빨라지는가?

### 알게 된 것

로그 스케일을 사용하면 저음 대역이 더 세밀하게 나뉜다. 실제 이퀄라이저와 스펙트럼 분석기가 모두 로그 스케일을 사용하는 이유가 여기 있다. 인간의 청각에 맞게 설계된 것이다.

## 최종 프로젝트. 16열 스펙트럼 분석기를 완성하라

### 설계

지금까지 만든 조각들을 합쳐 다음을 구현한다.

| 요소 | 설명 |
|------|------|
| 입력 | I2S 마이크, 초당 8000 샘플 |
| 주파수 대역 | 80 Hz ~ 900 Hz, 16등분 (로그 스케일) |
| 시각화 | 16 × 16 LED, 각 열이 해당 주파수의 세기 표시 |
| 색상 | 저음 파랑 → 고음 빨강 그라데이션 |

### 최종 코드

파일 이름: `spectrum.py`

```python
# spectrum.py
from machine import Pin, I2S
from ws2812 import WS2812Matrix
import struct, math

# ── 하드웨어 초기화 ───────────────────────────────
mic = I2S(
    0,
    sck=Pin(4), ws=Pin(5), sd=Pin(6),
    mode=I2S.RX, bits=32, format=I2S.MONO,
    rate=8000, ibuf=4096
)
m = WS2812Matrix([(0, 1)])

# ── 파라미터 ──────────────────────────────────────
N        = 256          # 한 프레임당 샘플 수
RATE     = 8000         # 샘플링 레이트 (Hz)
COLS     = 16           # 열 (주파수 대역) 수
ROWS     = 16           # 행 (LED 높이)
FREQ_MIN = 80.0
FREQ_MAX = 900.0
BRIGHT   = 0.4          # 전체 밝기 조절 (0.0~1.0)

buf = bytearray(N * 4)

# ── Goertzel 알고리즘 ─────────────────────────────
def goertzel(samples, freq, rate):
    n     = len(samples)
    k     = int(0.5 + n * freq / rate)
    omega = 2.0 * math.pi * k / n
    coeff = 2.0 * math.cos(omega)
    s1 = s2 = 0.0
    for x in samples:
        s0 = x + coeff * s1 - s2
        s2 = s1
        s1 = s0
    return max(0.0, s2 * s2 + s1 * s1 - coeff * s1 * s2)

# ── 로그 스케일 주파수 배열 ───────────────────────
def log_freqs(f_min, f_max, n):
    lmin = math.log(f_min)
    lmax = math.log(f_max)
    step = (lmax - lmin) / (n - 1)
    return [math.exp(lmin + i * step) for i in range(n)]

FREQS = log_freqs(FREQ_MIN, FREQ_MAX, COLS)

# ── 열 그리기 ─────────────────────────────────────
def draw_col(x, height):
    ratio = x / (COLS - 1)            # 0.0 (저음) ~ 1.0 (고음)
    r = int(255 * ratio   * BRIGHT)
    g = int(80  * (1 - abs(ratio - 0.5) * 2) * BRIGHT)
    b = int(255 * (1 - ratio) * BRIGHT)

    for y in range(ROWS):
        m[x, y].value = (0, 0, 0)
    for i in range(min(height, ROWS)):
        m[x, ROWS - 1 - i].value = (r, g, b)

# ── 메인 루프 ─────────────────────────────────────
print("스펙트럼 분석기 시작")

while True:
    mic.readinto(buf)
    raw    = struct.unpack('<' + 'i' * N, buf)
    vals   = [s >> 14 for s in raw]

    # 각 주파수 대역의 세기 계산
    powers = [goertzel(vals, f, RATE) for f in FREQS]
    mx     = max(powers + [1.0])

    for x, p in enumerate(powers):
        h = int((p / mx) * ROWS)
        draw_col(x, h)

    m.update()
```

```sh
replx spectrum.py
```
