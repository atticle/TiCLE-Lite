## GPIO

GPIO는 General Purpose Input Output의 약자로, 디지털 신호를 입력받거나 출력하는 범용 핀이다. LED 켜기/끄기, 스위치 입력 읽기처럼 가장 기본적인 제어는 대부분 GPIO에서 시작한다.

MicroPython에서는 `machine` 라이브러리의 `Pin` 클래스로 GPIO를 제어한다. 핀 번호와 모드(`Pin.OUT`, `Pin.IN`)를 지정해 원하는 동작을 만들 수 있다.

### LED

아래 사진과 같이 Core 보드에 위치한 LED를 제어한다.

<img src="pds/led.png" style="width:30%">

VSCode에서 아래와 같이 코드를 작성한 후 `led.py`라는 이름으로 저장한다.

```py
# led.py
from machine import Pin
import utime

led_pin = Pin('LED', Pin.OUT)

led_pin.on()
utime.sleep_ms(1000)
led_pin.off()
utime.sleep_ms(1000)

led_pin.value(1)
utime.sleep_ms(1000)
led_pin.value(0)
utime.sleep_ms(1000)
```

핵심은 `Pin('LED', Pin.OUT)` 한 줄이다. 첫 번째 인자는 제어할 핀이고, 두 번째 인자는 핀의 방향이다. LED를 켤 것이므로 출력 모드(`Pin.OUT`)를 사용한다.

예제에서는 `on()/off()`와 `value(1)/value(0)` 두 방법을 모두 보여 준다. 두 방식은 같은 결과를 만들며, `sleep_ms(1000)`으로 1초 대기를 넣어 점멸 주기를 조절한다.

터미널에 다음 명령어를 입력하여 프로그램을 실행시키면, 코어 보드에 부착된 LED가 점멸하는 것을 확인할 수 있다.

```sh

```

```python
# ultra.py
from machine import Pin, time_pulse_us
import time

trig = Pin(10, Pin.OUT)
echo = Pin(11, Pin.IN)

trig.value(0)
time.sleep_ms(100)

while True:
    trig.value(0)
    time.sleep_us(2)
    trig.value(1)
    time.sleep_us(10)
    trig.value(0)

    try:
        duration = time_pulse_us(echo, 1, 30000)
        distance = duration * 0.0343 / 2
        print(distance)
    except OSError:
        print("timeout")

    time.sleep_ms(200)
```

## PIO

PIO는 Programmable I/O의 약자로, 정밀한 타이밍이 필요한 신호를 하드웨어가 직접 처리하도록 돕는 기능이다. CPU가 모든 타이밍을 직접 맞추지 않아도 되어 안정성이 높아진다.

WS2812 같은 LED는 신호 타이밍이 매우 중요하므로, 일반 GPIO보다 PIO를 사용할 때 훨씬 안정적으로 동작한다.

### PixelLed

PixelLed(WS2812)는 한 줄의 데이터 라인으로 여러 LED를 연결해 제어할 수 있다. 다만 신호 폭이 매우 정확해야 하므로 소프트웨어 루프만으로는 깜빡임이나 오동작이 발생할 수 있다.

이 예제는 PIO로 신호를 만들어 LED에 데이터를 보낸다. 따라서 CPU 부하를 줄이면서도 안정적으로 색상을 표시할 수 있다.

아래 사진과 같이 위치한 PixelLed를 제어해 보자.

![PixelLed](pds/pixelled_pic.png)

| Peripheral | Pin number |
| ------ | ---------- |
| PixelLed | 0 |

VSCode에서 아래와 같이 코드를 작성한 후 `pixelled.py`라는 이름으로 저장한다.

```py
# pixelled.py
from machine import Pin
from rp2 import PIO, StateMachine, asm_pio
import utime

WIDTH, HEIGHT = 16, 16

@asm_pio(
    sideset_init=PIO.OUT_LOW,
    out_shiftdir=PIO.SHIFT_LEFT,
    autopull=True,
    pull_thresh=24
)
def pixelled():
    T1 = 2
    T2 = 5
    T3 = 3
    wrap_target()
    label("bitloop")
    out(x, 1)               .side(0) [T3 - 1]
    jmp(not_x, "do_zero")   .side(1) [T1 - 1]
    jmp("bitloop")          .side(1) [T2 - 1]
    label("do_zero")
    nop()                   .side(0) [T2 - 1]
    wrap()

class PixelLedMatrixPIO:
    def __init__(self, pin, width=WIDTH, height=HEIGHT, sm_id=0, freq=8_000_000):
        self.width, self.height = width, height
        self.buf = bytearray(width * height * 3)
        self.sm = StateMachine(sm_id, pixelled, freq=freq, sideset_base=Pin(pin))
        self.sm.active(1)

    def _idx(self, x, y):
        return (y * self.width + x) * 3

    def __setitem__(self, xy, rgb):
        x, y = xy
        r, g, b = rgb
        i = self._idx(x, y)
        self.buf[i + 0] = g & 0xFF
        self.buf[i + 1] = r & 0xFF
        self.buf[i + 2] = b & 0xFF

    def clear(self):
        for i in range(len(self.buf)):
            self.buf[i] = 0

    def show(self):
        mv = memoryview(self.buf)
        for i in range(0, len(mv), 3):
            v = (mv[i] << 16) | (mv[i + 1] << 8) | mv[i + 2]
            self.sm.put(v, 8)
        utime.sleep_us(300)

matrix = PixelLedMatrixPIO(pin=0, width=16, height=16)
matrix.clear()
matrix[3, 4] = (255, 0, 0)
matrix[4, 4] = (0, 255, 0)
matrix[5, 4] = (0, 0, 255)
matrix.show()
```

핵심 흐름은 간단하다. 버퍼에 좌표별 RGB 값을 넣고 `show()`를 호출하면, PIO가 정해진 타이밍으로 데이터를 전송해 LED에 색을 표시한다. 실행하면 지정한 위치의 픽셀이 빨강, 초록, 파랑으로 켜진다.

터미널에서 다음 명령어를 입력해 프로그램을 실행한다. 지정한 좌표의 픽셀이 지정한 색으로 켜진다.

```sh

```

### VR

## PWM

PWM은 디지털 신호의 ON 시간 비율(듀티비)을 바꿔 평균 출력 세기를 조절하는 방식이다. 쉽게 말해 아주 빠르게 켜고 끄는 비율을 바꿔 밝기나 속도를 제어한다.

PWM의 핵심은 주파수와 듀티비다. 예를 들어 듀티비가 0%면 항상 꺼진 상태, 100%면 항상 켜진 상태, 50%면 절반 세기로 동작하는 효과를 낸다.

MicroPython에서는 `machine` 라이브러리의 `PWM` 클래스를 사용한다.

### Servo Motor

아래 사진과 같이 위치한 서보 모터를 제어해 보자.

<img src="pds/servo.png" style="width:50%;">

| Peripheral | Pin number |
| ------ | ---------- |
| Servo | 1 |

VSCode에서 아래와 같이 코드를 작성한 후 `servo.py`라는 이름으로 저장한다.

```py
# servo.py
from machine import Pin, PWM
import utime

class Servo:
    def __init__(self, pin, freq=50):
        self.pwm = PWM(Pin(pin))
        self.pwm.freq(freq)
        self.min_duty = 1638
        self.max_duty = 8191

    def write(self, angle):
        angle = max(0, min(180, angle))
        duty = self.min_duty + (self.max_duty - self.min_duty) * angle // 180
        self.pwm.duty_u16(duty)

    def deinit(self):
        self.pwm.deinit()

servo = Servo(pin=1)
for ang in range(0, 181, 30):
    servo.write(ang)
    utime.sleep(0.5)
for ang in range(180, -1, -30):
    servo.write(ang)
    utime.sleep(0.5)
servo.deinit()
```

초기화 시 PWM 주파수를 50Hz로 설정하며, `write(angle)` 함수로 0~180도 범위의 각도를 듀티비로 변환해 제어한다.

`deinit()` 함수는 PWM 출력을 종료해 서보모터를 멈춘다. 예제에서는 0도에서 180도까지 30도 간격으로 움직인 뒤 원위치로 돌아오는 동작을 반복한다.

터미널에서 다음 명령어를 입력해 프로그램을 실행한다. 서보모터가 0~180도 범위에서 30도 간격으로 회전하는 것을 확인할 수 있다.


## I2C

I2C는 두 선(SCL, SDA)으로 여러 장치를 연결해 통신하는 방식이다. 각 장치는 고유 주소를 가지므로, 같은 버스에 여러 장치를 붙여도 구분해서 제어할 수 있다.

TiCLE Lite에서 I2C 핀 연결 정보는 다음과 같다.

| Connection pin | Pin number |
| ------ | ---------- |
| SCL | 13 |
| SDA | 12 |

### LCD

아래 사진과 같이 위치한 LCD를 제어해 보자. LCD 모듈 명칭은 `PCF8574`이며, 데이터시트는 다음 링크에서 확인할 수 있다.
- https://www.alldatasheet.com/datasheet-pdf/pdf/18212/PHILIPS/PCF8574.html

<img src="pds/lcd_pic.png" style="width:50%;">

<br>

VSCode에서 아래와 같이 코드를 작성한 후 `lcd.py`라는 이름으로 저장한다.

```python
# lcd.py
from machine import I2C, Pin
import utime

ADDR = 0x27

class LCD_PCF8574:
    def __init__(self, i2c, addr=ADDR, backlight=True):
        self.i2c = i2c
        self.addr = addr
        self.bl = 0x08 if backlight else 0x00
        utime.sleep_ms(50)
        self._pulse(0x30)
        utime.sleep_ms(5)
        self._pulse(0x30)
        utime.sleep_us(200)
        self._pulse(0x30)
        self._pulse(0x20)
        self.cmd(0x28)
        self.cmd(0x0C)
        self.cmd(0x06)
        self.clear()

    def _send_raw(self, data):
        self.i2c.writeto(self.addr, bytes([data | self.bl]))

    def _pulse(self, data):
        self._send_raw(data | 0x04)
        utime.sleep_us(500)
        self._send_raw(data & ~0x04)
        utime.sleep_us(100)

    def _send_nibble(self, nibble, rs):
        data = ((nibble & 0x0F) << 4) | (0x01 if rs else 0x00)
        self._send_raw(data)
        self._pulse(data)

    def _send_byte(self, val, rs):
        self._send_nibble(val >> 4, rs)
        self._send_nibble(val & 0x0F, rs)

    def cmd(self, c):
        self._send_byte(c, 0)
        utime.sleep_ms(2)

    def write(self, ch):
        self._send_byte(ch, 1)

    def clear(self):
        self.cmd(0x01)
        utime.sleep_ms(2)

    def text(self, s):
        for ch in s:
            self.write(ord(ch))

i2c = I2C(0, scl=Pin(13), sda=Pin(12), freq=100000)
lcd = LCD_PCF8574(i2c, ADDR)
lcd.text("hi")
utime.sleep_ms(2000)
lcd.clear()
```

I2C 통신에서는 장치 주소가 필요하다. 본 예제는 코드와 동일하게 LCD 주소를 `0x27`로 사용한다. 일부 모듈은 `0x3F`를 사용할 수 있으므로, 동작하지 않으면 I2C 스캔으로 주소를 먼저 확인한다.

프로그램은 I2C 객체를 만든 뒤 `LCD_PCF8574` 객체를 생성한다. 이 과정에서 LCD 초기화 명령이 전송되며, `text()`로 문자열을 쓰고 `clear()`로 화면을 지운다.

정확한 명령어와 타이밍은 위 데이터시트를 참고한다.

터미널에서 다음 명령어를 입력해 프로그램을 실행한다. LCD에 'hi'가 출력되고 2초 후 화면이 초기화된다.

```sh

```

### IMU

```python
# imu.py
from machine import I2C, Pin
import utime, struct

MPU_ADDR=0x68
REG_PWR=0x6B
REG_ACCEL=0x3B
REG_GYRO_CFG=0x1B
REG_ACC_CFG=0x1C

i2c=I2C(0, scl=Pin(13), sda=Pin(12), freq=400000)

def wreg(r,v):
    i2c.writeto_mem(MPU_ADDR,r,bytes([v]))

def init():
    wreg(REG_PWR,0x00)
    wreg(REG_ACC_CFG,0x00)
    wreg(REG_GYRO_CFG,0x00)
    utime.sleep_ms(50)

def read_raw():
    buf=bytearray(14)
    i2c.readfrom_mem_into(MPU_ADDR,REG_ACCEL,buf)
    ax,ay,az,tmp,gx,gy,gz=struct.unpack(">hhhhhhh",buf)
    return ax,ay,az,gx,gy,gz

def read():
    ax,ay,az,gx,gy,gz=read_raw()
    ax/=16384.0; ay/=16384.0; az/=16384.0
    gx/=131.0;   gy/=131.0;   gz/=131.0
    return (ax,ay,az),(gx,gy,gz)

init()

while True:
    acc,gyro=read()
    print("ACC",acc,"GYR",gyro)
    utime.sleep_ms(100)
```

## I2S

I2S는 디지털 오디오 전송을 위한 통신 방식이다. 마이크 입력이나 스피커 출력처럼 "끊기면 안 되는" 소리 데이터를 일정한 속도로 주고받는 데 적합하다.

I2S는 주로 비트 클럭(BCLK), 좌우 클럭(LRCLK), 데이터선(SD)을 사용한다. 이 구조 덕분에 오디오 데이터를 안정적으로 전송할 수 있다.

핀 연결 정보는 다음과 같다.

| Connection pin | Pin number |
| ------ | ---------- |
| BCL | 4 |
| LRCL | 5 |

```python
# mic.py
from machine import Pin, I2S
import math
import struct
import time

I2S_ID = 0
SAMPLE_RATE = 16000
BITS = 32
BUFFER_BYTES = 4096

SCK_PIN = 4
WS_PIN  = 5
MIC_PIN  = 6
audio_in = I2S(
    I2S_ID,
    sck=Pin(SCK_PIN),
    ws=Pin(WS_PIN),
    sd=Pin(MIC_PIN),
    mode=I2S.RX,
    bits=BITS,
    format=I2S.MONO,  
    rate=SAMPLE_RATE,
    ibuf=BUFFER_BYTES
)

def calc_rms_db(samples):
    if not samples:
        return -120.0

    acc = 0
    for s in samples:
        acc += s * s
    mean_sq = acc / len(samples)
    rms = math.sqrt(mean_sq)

    if rms == 0:
        return -120.0  

    full_scale = (1 << 31) - 1
    db = 20 * math.log10(rms / full_scale)
    return db

FRAME_BYTES = 4  
READ_BYTES = 1024 

buf = bytearray(READ_BYTES)

try:
    while True:
        read_len = audio_in.readinto(buf)
        if read_len is None or read_len == 0:
            time.sleep(0.01)
            continue

        n_frames = read_len // FRAME_BYTES

        samples = []
        for i in range(n_frames):
            b0 = buf[4*i + 0]
            b1 = buf[4*i + 1]
            b2 = buf[4*i + 2]
            b3 = buf[4*i + 3]
            s = struct.unpack('<i', bytes((b0, b1, b2, b3)))[0]

            samples.append(s)

        db = calc_rms_db(samples)
        print("Level: {:.1f} dBFS".format(db))

        time.sleep(0.05)

except KeyboardInterrupt:
    pass

finally:
    audio_in.deinit()
```

```python
# speaker.py
from machine import Pin, I2S

I2S_ID = 0
WAV_PATH = "test.wav"

BCK_PIN = 4 
LRCK_PIN = 5
SPEAKER_PIN = 7

SAMPLE_RATE = 16000
BITS = 16
BUFFER_SIZE = 2048

audio_out = I2S(
    I2S_ID,
    sck=Pin(BCK_PIN),
    ws=Pin(LRCK_PIN),
    sd=Pin(SPEAKER_PIN),
    mode=I2S.TX,
    bits=BITS,
    format=I2S.MONO,
    rate=SAMPLE_RATE,
    ibuf=BUFFER_SIZE,
)

buf = bytearray(BUFFER_SIZE)

with open(WAV_PATH, "rb") as f:
    f.read(44)
    while True:
        n = f.readinto(buf)
        if n == 0:
            break
        written = 0
        while written < n:
            written += audio_out.write(buf[written:n])

audio_out.deinit()
```
