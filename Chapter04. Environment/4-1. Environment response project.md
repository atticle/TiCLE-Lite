# 환경 반응형 LED 매트릭스 프로젝트

## 학습 목표

- VR, CdS, 마이크 센서값을 읽고 원하는 범위의 숫자로 변환할 수 있다.
- 숫자를 색상, 밝기, 크기와 같은 시각 요소에 연결하는 설계를 할 수 있다.
- 세 가지 센서를 하나의 LED 화면에서 동시에 반응하도록 만들 수 있다.

---

## 이 장의 도전 문제

> 방 안의 분위기가 바뀔 때마다 조명이 자동으로 반응한다면 어떨까?
> 다이얼을 돌리면 테두리 색이 바뀌고, 주변이 어두워지면 화면이 밝아지고, 박수를 치면 가운데 원이 커진다.
> 이걸 직접 만들 수 있을까?

처음부터 이 전체를 한꺼번에 만들려고 하지 않는다. 가장 작은 문제부터 하나씩 해결한다. 미션을 하나씩 완성하다 보면, 마지막 미션이 끝났을 때 최종 프로젝트가 자연스럽게 완성된다.

---

## 사용하는 장치

| 장치 | 핀 | 역할 |
|------|-----|------|
| LED 매트릭스 | 핀 0 (SM 1) | 16×16 픽셀 출력 |
| 가변저항 (VR) | ADC 핀 27 | 다이얼을 돌려 0~65535 숫자 출력 |
| 조도센서 (CdS) | ADC 핀 28 | 밝을수록 큰 숫자 출력 |
| 마이크 | SCK=4, WS=5, SD=6 | 소리의 진동을 디지털 숫자 배열로 출력 |

## 미션 1. LED 픽셀 하나를 내 마음대로 켜 보자

### 문제

16×16 LED 매트릭스에서 원하는 위치에 원하는 색으로 픽셀 하나를 켜라.

### 생각해 보기

TV 화면을 아주 가까이서 보면 아주 작은 점들이 빨강, 초록, 파랑으로 빛나는 것을 볼 수 있다. 이 LED 매트릭스도 같은 원리로 동작한다. 가로 16칸, 세로 16칸의 격자에서 픽셀 하나를 켜려면 두 가지를 정해야 한다.

- **위치**: `(x, y)` 좌표 — 왼쪽 위가 `(0, 0)`, 오른쪽 아래가 `(15, 15)`이다.
- **색**: `(R, G, B)` — 빨강, 초록, 파랑의 양을 각각 0(꺼짐)에서 255(최대)로 정한다.

### 코드

파일 이름: `m1_one_pixel.py`

```python
# m1_one_pixel.py
from ws2812 import WS2812Matrix
import utime

m = WS2812Matrix([(0, 1)])   # 핀 0, 상태 머신 1
m.clear()                    # 모든 픽셀 끄기

m[8, 8].value = (255, 0, 0)  # (8, 8) 위치에 빨간색
m.update()                   # 화면에 반영
utime.sleep_ms(1500)

m[8, 8].value = (0, 255, 0)  # 같은 자리에 초록색
m.update()
utime.sleep_ms(1500)

m[8, 8].value = (0, 0, 255)  # 파란색
m.update()
```

```sh
replx m1_one_pixel.py
```

### 관찰하고 실험해 보자

아래 값들을 바꿔 가며 다시 실행해 보자.

- `m[8, 8]`을 `m[0, 0]`, `m[15, 0]`, `m[15, 15]`로 바꾸면 점이 어디에 켜지는가?
- `(255, 0, 0)` 대신 `(255, 255, 0)`, `(0, 255, 255)`, `(255, 0, 255)`를 써 보자. 무슨 색이 나오는가?
- `(255, 0, 0)` 대신 `(50, 0, 0)` 또는 `(10, 0, 0)`으로 줄이면 어떻게 달라지는가?

### 알게 된 것

모든 픽셀은 `(x, y)` 좌표와 `(R, G, B)` 색으로 완전히 표현된다. 이것이 컴퓨터가 색과 위치를 다루는 가장 기본적인 방식이다.

## 미션 2. 다이얼을 돌리면 어떤 숫자가 나오는가

### 문제

가변저항(VR) 다이얼을 돌렸을 때 어떤 숫자가 나오는지 읽어 터미널에 출력하라.

### 생각해 보기

가변저항은 다이얼을 돌릴수록 내부 저항 값이 바뀐다. 보드의 ADC 회로는 이 저항 변화를 감지해 0에서 65535 사이의 숫자로 바꿔 준다.

65535라는 숫자가 왜 나오는지 생각해 보자. 이 보드의 ADC는 16비트(bit) 해상도이다. 1비트는 0 또는 1 두 가지 상태를 나타낸다. 16비트는 $2^{16} = 65536$가지를 표현할 수 있다. 0부터 시작하므로 최대값이 65535가 된다. 비트가 많을수록 더 세밀하게 나눌 수 있다. 이것을 **해상도**라고 한다.

### 코드

파일 이름: `m2_vr_read.py`

```python
# m2_vr_read.py
from machine import Pin, ADC
import utime

vr = ADC(Pin(27))   # 핀 27번에 연결된 가변저항

while True:
    val = vr.read_u16()   # 0 ~ 65535
    print("VR:", val)
    utime.sleep_ms(200)
```

```sh
replx m2_vr_read.py
```

### 관찰하고 실험해 보자

- 다이얼을 한쪽 끝까지 돌렸을 때 최솟값은 얼마인가? 반대쪽 끝은?
- 다이얼을 정확히 가운데에 두면 65535의 절반인 약 32767에 가까운가?
- 다이얼을 고정하고 숫자가 완전히 멈추는가, 아니면 조금씩 흔들리는가? 왜 그럴까?

### 알게 된 것

물리적인 변화(저항이 바뀜)가 숫자로 바뀌는 것이 ADC(아날로그-디지털 변환)의 핵심이다. 이 숫자 하나로 이후 LED, 서보모터 등 모든 것을 제어할 수 있다.

## 미션 3. 숫자를 LED 밝기로 변환하라

### 문제

다이얼을 돌렸을 때 LED 픽셀 한 개의 빨간색 밝기가 따라 변하도록 만들어라.

### 생각해 보기

미션 2에서 VR이 0~65535 범위의 숫자를 준다는 것을 알았다. 그런데 LED 색상 값은 0~255 범위이다. 두 범위를 연결하는 가장 간단한 방법은 나눗셈이다.

$$\text{밝기} = \left\lfloor \frac{\text{VR 값}}{256} \right\rfloor$$

예를 들어 VR이 65280이면 $65280 \div 256 = 255$이다. VR이 0이면 밝기도 0, 최대이면 255가 된다.

이처럼 **한 범위의 값을 다른 범위로 바꾸는 것**을 **매핑(mapping)** 이라고 한다. 센서 프로그래밍에서 가장 자주 쓰이는 개념이다.

### 코드

파일 이름: `m3_vr_brightness.py`

```python
# m3_vr_brightness.py
from ws2812 import WS2812Matrix
from machine import Pin, ADC
import utime

m  = WS2812Matrix([(0, 1)])
vr = ADC(Pin(27))

while True:
    val    = vr.read_u16()          # 0 ~ 65535
    bright = val // 256             # 0 ~ 255 로 변환 (정수 나눗셈)
    m.clear()
    m[8, 8].value = (bright, 0, 0)  # 빨간색, 밝기만 변함
    m.update()
    utime.sleep_ms(50)
```

```sh
replx m3_vr_brightness.py
```

### 관찰하고 실험해 보자

- 다이얼을 끝까지 돌리면 LED가 가장 밝아지는가?
- `val // 256` 대신 `val // 512`로 바꾸면 밝기 변화가 어떻게 달라지는가? 왜 그럴까?
- 픽셀 `m[8, 8]` 하나가 아니라 여러 좌표를 같은 방법으로 켜려면 어떻게 바꿔야 할까?

### 알게 된 것

센서 데이터를 그대로 쓰는 경우는 거의 없다. 항상 원하는 범위로 변환(매핑)이 필요하다. `//`는 정수 나눗셈(소수점 버림)이다. 색상 값처럼 정수만 받는 곳에서는 반드시 정수로 변환해야 한다.

## 미션 4. 다이얼로 색을 무지개처럼 바꿔라

### 문제

다이얼을 천천히 돌리면 LED 색이 빨강 → 주황 → 노랑 → 초록 → 파랑 → 보라 → 빨강 순서로 부드럽게 바뀌도록 만들어라.

### 생각해 보기

RGB 방식으로 무지개 색을 만들려면 R, G, B 세 값을 동시에 계산해야 해서 복잡하다. 그러나 **HSV** 방식을 쓰면 훨씬 쉽다.

| 요소 | 의미 | 값 범위 |
|------|------|---------|
| H (Hue, 색조) | 색의 종류 | 0.0~1.0 (0.0=빨강, 0.33=초록, 0.67=파랑, 1.0=다시 빨강) |
| S (Saturation, 채도) | 색의 선명함 | 0.0(회색)~1.0(가장 선명) |
| V (Value, 명도) | 색의 밝기 | 0.0(검정)~1.0(가장 밝음) |

H 값 하나만 0.0에서 1.0으로 바꿔 가면 무지개 전체를 표현할 수 있다. S와 V를 고정하면 색의 선명함과 밝기는 그대로 유지된다.

### 코드

파일 이름: `m4_vr_hue.py`

```python
# m4_vr_hue.py
from ws2812 import WS2812Matrix
from machine import Pin, ADC
import utime

m  = WS2812Matrix([(0, 1)])
vr = ADC(Pin(27))

def hsv_to_rgb(h, s, v):
    """HSV 값(각 0.0~1.0)을 RGB(각 0~255)로 변환한다."""
    if s == 0:
        c = int(v * 255)
        return (c, c, c)
    h6 = h * 6.0
    i  = int(h6) % 6
    f  = h6 - int(h6)
    p  = v * (1.0 - s)
    q  = v * (1.0 - s * f)
    t  = v * (1.0 - s * (1.0 - f))
    tbl = [(v,t,p), (q,v,p), (p,v,t), (p,q,v), (t,p,v), (v,p,q)]
    r, g, b = tbl[i]
    return (int(r * 255), int(g * 255), int(b * 255))

while True:
    hue = vr.read_u16() / 65535.0        # 0.0 ~ 1.0 으로 변환
    r, g, b = hsv_to_rgb(hue, 1.0, 0.5)
    m.clear()
    m[8, 8].value = (r, g, b)
    m.update()
    utime.sleep_ms(50)
```

```sh
replx m4_vr_hue.py
```

### 관찰하고 실험해 보자

- 다이얼을 천천히 돌려 보자. 색이 어떤 순서로 바뀌는가?
- `hsv_to_rgb(hue, 1.0, 0.5)`에서 세 번째 인자를 `1.0`이나 `0.1`로 바꾸면 어떻게 달라지는가?
- 채도(두 번째 인자)를 `0.0`으로 바꾸면 어떤 색이 나오는가? 왜일까?

### 알게 된 것

같은 색이라도 표현 방식이 여러 가지이다. RGB는 색을 만드는 재료의 양으로, HSV는 색의 종류·선명함·밝기로 나타낸다. 색이 연속으로 바뀌는 효과를 만들 때는 HSV가 훨씬 편하다.

함수는 복잡한 계산을 이름 하나 뒤에 숨기는 도구이다. `hsv_to_rgb`를 쓸 때 내부 계산을 몰라도 된다. 이름과 입출력만 알면 된다.

## 미션 5. 테두리를 같은 색으로 물들여라

### 문제

다이얼을 돌릴 때 중앙 한 점이 아니라 화면 가장자리 테두리 전체가 같은 색으로 바뀌도록 만들어라.

### 생각해 보기

16×16 격자에서 테두리 픽셀의 특징은 무엇인가?

- x가 0이거나 15인 열 (왼쪽 끝, 오른쪽 끝)
- y가 0이거나 15인 행 (위쪽 끝, 아래쪽 끝)

모든 픽셀을 반복문으로 하나씩 확인하면서, 위 조건에 맞으면 색을 칠하고 아니면 끄면 된다. 전체 픽셀은 16 × 16 = 256개이므로, 이중 반복문이 256번 실행된다.

### 코드

파일 이름: `m5_border.py`

```python
# m5_border.py
from ws2812 import WS2812Matrix
from machine import Pin, ADC
import utime

m  = WS2812Matrix([(0, 1)])
vr = ADC(Pin(27))

def hsv_to_rgb(h, s, v):
    if s == 0:
        c = int(v * 255); return (c, c, c)
    h6 = h * 6.0; i = int(h6) % 6; f = h6 - int(h6)
    p = v*(1-s); q = v*(1-s*f); t = v*(1-s*(1-f))
    tbl = [(v,t,p),(q,v,p),(p,v,t),(p,q,v),(t,p,v),(v,p,q)]
    r, g, b = tbl[i]
    return (int(r*255), int(g*255), int(b*255))

W, H = 16, 16

while True:
    hue = vr.read_u16() / 65535.0
    r, g, b = hsv_to_rgb(hue, 1.0, 0.5)

    for y in range(H):
        for x in range(W):
            if x == 0 or x == W-1 or y == 0 or y == H-1:
                m[x, y].value = (r, g, b)   # 테두리 픽셀
            else:
                m[x, y].value = (0, 0, 0)   # 나머지는 끔
    m.update()
    utime.sleep_ms(50)
```

```sh
replx m5_border.py
```

### 관찰하고 실험해 보자

- 테두리 조건을 `x == 1 or x == W-2 or y == 1 or y == H-2`로 바꾸면 어떻게 보이는가?
- 테두리를 두 줄로 만들려면 조건을 어떻게 써야 할까?
- `or` 네 개를 모두 `and`로 바꾸면 어느 픽셀이 켜지는가? 왜 그럴까?

### 알게 된 것

이중 반복문(`for y` 안에 `for x`)으로 모든 픽셀을 순서대로 방문한다. 조건문으로 어떤 픽셀을 칠할지 골라낸다. **반복문 + 조건문** 조합이 컴퓨터 그래픽의 기본 방식이다.

## 미션 6. 주변이 어두울수록 화면이 밝아지게 하자

### 문제

CdS 조도센서를 이용해 주변이 어두울수록 LED 화면이 더 밝아지고, 주변이 밝을수록 화면이 어두워지도록 만들어라.

### 생각해 보기

CdS 센서는 빛이 강할수록 큰 숫자를 출력한다. 그런데 우리 목표는 반대다. 어두우면 LED가 밝아야 한다.

숫자를 뒤집는 방법은 간단하다.

1. CdS 값을 0.0~1.0으로 **정규화**: `cds_norm = cds.read_u16() / 65535.0`
2. **뒤집기**: `brightness = 1.0 - cds_norm`

| 상황 | CdS 값 | cds_norm | brightness | LED |
|------|--------|----------|------------|-----|
| 강한 빛 | ~65535 | ~1.0 | ~0.0 | 거의 꺼짐 |
| 보통 | ~32767 | ~0.5 | ~0.5 | 중간 |
| 어둠 | ~0 | ~0.0 | ~1.0 | 최대 밝음 |

**정규화(normalization)** 란 값을 0.0~1.0 범위로 맞추는 것이다. 서로 단위가 다른 센서들을 같은 기준으로 계산할 때 쓰인다.

### 코드

파일 이름: `m6_cds_brightness.py`

```python
# m6_cds_brightness.py
from ws2812 import WS2812Matrix
from machine import Pin, ADC
import utime

m   = WS2812Matrix([(0, 1)])
vr  = ADC(Pin(27))
cds = ADC(Pin(28))

def hsv_to_rgb(h, s, v):
    if s == 0:
        c = int(v * 255); return (c, c, c)
    h6 = h * 6.0; i = int(h6) % 6; f = h6 - int(h6)
    p = v*(1-s); q = v*(1-s*f); t = v*(1-s*(1-f))
    tbl = [(v,t,p),(q,v,p),(p,v,t),(p,q,v),(t,p,v),(v,p,q)]
    r, g, b = tbl[i]
    return (int(r*255), int(g*255), int(b*255))

W, H = 16, 16

while True:
    hue        = vr.read_u16()  / 65535.0   # VR  → 색조
    cds_norm   = cds.read_u16() / 65535.0   # CdS → 0.0(어둠) ~ 1.0(밝음)
    brightness = 1.0 - cds_norm             # 뒤집기: 어두울수록 밝게

    r, g, b = hsv_to_rgb(hue, 1.0, brightness)

    for y in range(H):
        for x in range(W):
            if x == 0 or x == W-1 or y == 0 or y == H-1:
                m[x, y].value = (r, g, b)
            else:
                m[x, y].value = (0, 0, 0)
    m.update()
    utime.sleep_ms(50)
```

```sh
replx m6_cds_brightness.py
```

### 관찰하고 실험해 보자

- CdS 센서를 손으로 가리면 LED 테두리가 밝아지는가?
- 밝은 손전등을 CdS에 비추면 LED가 거의 꺼지는가?
- `1.0 - cds_norm` 을 그냥 `cds_norm`으로 바꾸면 동작이 어떻게 달라지는가?

### 알게 된 것

센서값을 `÷ 최대값`으로 나누면 0.0~1.0으로 정규화된다. `1.0 - x`로 뒤집으면 크고 작음이 반대가 된다. 이 두 가지 변환은 센서 프로그래밍에서 매우 자주 쓰이는 패턴이다.

## 미션 7. 소리를 그림으로 바꿔라

### 문제

마이크에 소리를 내면 화면 가운데에 원이 커지고, 조용해지면 원이 작아지도록 만들어라.

### 생각해 보기

마이크는 공기의 진동을 전기 신호로 바꾼다. 보드는 이 신호를 초당 16000번 측정해 숫자 배열로 저장한다. 소리가 크면 숫자들이 크게 흔들리고, 조용하면 0 근처에 모인다.

이 "흔들림의 정도"를 숫자 하나로 표현하는 방법이 **RMS(Root Mean Square)** 이다.

$$\text{RMS} = \sqrt{\frac{x_1^2 + x_2^2 + \cdots + x_n^2}{n}}$$

왜 제곱을 먼저 할까? 소리 신호는 양수와 음수를 번갈아 오가기 때문에 그냥 평균을 내면 0이 된다. 제곱하면 모두 양수가 되어 의미 있는 크기를 계산할 수 있다.

**원 그리기**: 중심 `(cx, cy)`에서 픽셀 `(x, y)`까지 거리가 반지름 r 이하이면 원 안에 있다.

$$(x - cx)^2 + (y - cy)^2 \leq r^2$$

제곱근 계산을 생략해 속도를 높인 형태이다.

### 코드

파일 이름: `m7_mic_circle.py`

```python
# m7_mic_circle.py
from ws2812 import WS2812Matrix
from machine import Pin, I2S
import utime, math, struct

m   = WS2812Matrix([(0, 1)])
mic = I2S(0, sck=Pin(4), ws=Pin(5), sd=Pin(6),
          mode=I2S.RX, bits=32, format=I2S.MONO,
          rate=16000, ibuf=4096)

buf    = bytearray(1024)
W, H   = 16, 16
cx, cy = 7, 7   # 원의 중심

# ── 이 값을 먼저 조정한다 ─────────────────────────
# 실행 후 터미널에 찍히는 RMS 숫자를 확인한 뒤,
# 박수를 쳤을 때 최대값을 MAX_RMS에 입력한다.
MAX_RMS = 5_000_000
# ──────────────────────────────────────────────────

def calc_rms(data, n_bytes):
    n = n_bytes // 4
    if n == 0:
        return 0.0
    total = 0
    for i in range(n):
        s = struct.unpack_from('<i', data, i * 4)[0]
        total += s * s
    return math.sqrt(total / n)

while True:
    n   = mic.readinto(buf)
    rms = calc_rms(buf, n)
    print("RMS:", int(rms))    # ← 처음 실행 때 이 값로 MAX_RMS를 맞춘다

    ratio  = min(1.0, rms / MAX_RMS)
    radius = max(1, int(1 + ratio * 5))   # 1 ~ 6

    for y in range(H):
        for x in range(W):
            dx, dy = x - cx, y - cy
            if dx*dx + dy*dy <= radius*radius:
                m[x, y].value = (0, 200, 60)   # 원 안: 초록
            else:
                m[x, y].value = (0, 0, 0)       # 원 밖: 꺼짐
    m.update()
```

```sh
replx m7_mic_circle.py
```

> **MAX_RMS 조정 방법**: 처음 실행하면 터미널에 RMS 숫자가 찍힌다. 조용할 때의 값과 박수를 쳤을 때의 최대값을 확인하고 `MAX_RMS`를 그 최대값 근처로 설정한다. 조정 후 `print(...)` 줄을 지우면 화면 갱신이 더 부드러워진다.

### 관찰하고 실험해 보자

- 박수를 치거나 마이크에 크게 말하면 원이 커지는가?
- 완전히 조용히 있으면 원이 최소 크기(1)로 줄어드는가?
- `cx, cy = 7, 7`을 `0, 0`으로 바꾸면 원이 어디를 중심으로 그려지는가?
- 원 색을 소리 크기에 따라 바꾸려면 어떻게 바꿔야 할까?

### 알게 된 것

모든 신호 처리는 "복잡한 입력을 원하는 숫자 하나로 요약하는 과정"이다. RMS는 흔들리는 소리 파형을 단 하나의 세기 숫자로 요약한다. 원 그리기는 수학의 거리 공식을 픽셀 조건으로 바꾼 것이다.

## 최종 프로젝트. 세 가지 센서를 하나의 화면에 담아라

지금까지 미션을 통해 각 센서를 어떻게 다루는지 배웠다.

| 미션 | 해결한 문제 | 핵심 개념 |
|------|-----------|----------|
| 1 | 픽셀 한 개 켜기 | RGB 색상, 좌표계 |
| 2 | VR 센서값 읽기 | ADC, 아날로그-디지털 변환 |
| 3 | 숫자를 밝기로 바꾸기 | 매핑, 범위 변환 |
| 4 | 색을 무지개로 바꾸기 | HSV 색공간, 함수 |
| 5 | 테두리 그리기 | 반복문 + 조건문으로 도형 만들기 |
| 6 | CdS로 밝기 조절 | 정규화, 값 뒤집기 |
| 7 | 소리로 원 크기 조절 | RMS, 원의 방정식 |

이제 이것을 하나로 합칠 차례이다.

### 설계

| 센서 | 역할 |
|------|------|
| VR (다이얼) | 테두리 색의 색조(H) 결정 |
| CdS (조도센서) | 테두리 색의 밝기(V) 결정 — 어두울수록 밝게 |
| 마이크 | 가운데 원의 반지름 — 소리가 클수록 크게 |

### 최종 프로젝트 코드

파일 이름: `env_project.py`

```python
# env_project.py
from ws2812 import WS2812Matrix
from machine import Pin, ADC, I2S
import utime, math, struct

# ── 장치 초기화 ─────────────────────────────────────
m   = WS2812Matrix([(0, 1)])
vr  = ADC(Pin(27))
cds = ADC(Pin(28))
mic = I2S(0, sck=Pin(4), ws=Pin(5), sd=Pin(6),
          mode=I2S.RX, bits=32, format=I2S.MONO,
          rate=16000, ibuf=4096)

buf     = bytearray(1024)
W, H    = 16, 16
cx, cy  = 7, 7
MAX_RMS = 5_000_000  # 미션 7 참고: 실험 후 조정

# ── 함수 정의 ────────────────────────────────────────
def hsv_to_rgb(h, s, v):
    if s == 0:
        c = int(v * 255); return (c, c, c)
    h6 = h * 6.0; i = int(h6) % 6; f = h6 - int(h6)
    p = v*(1-s); q = v*(1-s*f); t = v*(1-s*(1-f))
    tbl = [(v,t,p),(q,v,p),(p,v,t),(p,q,v),(t,p,v),(v,p,q)]
    r, g, b = tbl[i]
    return (int(r*255), int(g*255), int(b*255))

def calc_rms(data, n_bytes):
    n = n_bytes // 4
    if n == 0: return 0.0
    total = 0
    for i in range(n):
        s = struct.unpack_from('<i', data, i * 4)[0]
        total += s * s
    return math.sqrt(total / n)

# ── 메인 루프 ────────────────────────────────────────
while True:
    # 1. 센서 읽기 ─────────────────────────────
    hue        = vr.read_u16()  / 65535.0       # 색조
    cds_norm   = cds.read_u16() / 65535.0       # 주변 밝기
    brightness = 1.0 - cds_norm                 # 뒤집기

    n_bytes = mic.readinto(buf)
    rms     = calc_rms(buf, n_bytes)
    radius  = max(1, int(1 + min(1.0, rms / MAX_RMS) * 5))  # 1 ~ 6

    # 2. 색 계산 ──────────────────────────────
    r, g, b = hsv_to_rgb(hue, 1.0, brightness)

    # 3. 화면 그리기 ──────────────────────────
    for y in range(H):
        for x in range(W):
            is_border = (x == 0 or x == W-1 or y == 0 or y == H-1)
            dx, dy    = x - cx, y - cy
            is_circle = (dx*dx + dy*dy <= radius*radius)

            if is_border:
                m[x, y].value = (r, g, b)       # 테두리: VR 색, CdS 밝기
            elif is_circle:
                m[x, y].value = (0, 200, 60)    # 원: 초록
            else:
                m[x, y].value = (0, 0, 0)       # 나머지: 꺼짐
    m.update()
```

